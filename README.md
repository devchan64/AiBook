# AiBook

AI를 다시 학습하기 위한 개인 커리큘럼 북입니다.

배포 페이지: [https://devchan64.github.io/AiBook/](https://devchan64.github.io/AiBook/)

이 저장소는 대학 시절 AI 개론과 기초 과목, 프로젝트를 통해 배웠지만 시간이 지나 망각한 내용을 다시 복구하고, LLM과 생성형 AI로 대표되는 새로운 패러다임에 적응하기 위해 만든 정적 웹 책 프로젝트입니다.

## 목적

이 책은 흩어진 자료를 단순히 모으는 저장소가 아닙니다. 먼저 커리큘럼의 기준을 잡고, 그 기준에 맞춰 부가적인 자료를 수집하고 정리해가며 AI를 다시 이해하기 위한 저장소입니다.

다루려는 큰 흐름은 다음과 같습니다.

- 잊어버린 AI, 머신러닝, 딥러닝 기초 복구
- 수학, Python, 데이터 처리 기반 정리
- 전통적 머신러닝에서 딥러닝으로 넘어가는 사고 방식 정리
- Transformer, LLM, 생성형 AI, RAG, Agent 등 새로운 흐름 학습
- 학습 내용을 프로젝트로 검증
- 개인적인 이해와 표준적인 설명 사이의 차이 찾기

## AI로 만들어지는 책

이 책은 순전히 AI 도구를 통해 만들어지고 있습니다. 사람은 방향과 기준을 제시하고, AI는 커리큘럼 구성, 자료 정리, 초안 작성, 다이어그램 작성, 문서 구조화를 돕습니다.

따라서 이 책에는 틀린 내용, 불완전한 설명, 과도하게 단순화된 해석이 포함될 수 있습니다. 특히 AI가 생성한 설명은 그럴듯해 보여도 검증되지 않았을 수 있습니다. 잘못된 내용은 사람이 찾아서 정정해야 하며, 이 저장소는 그 과정을 기록하기 위한 공간이기도 합니다.

이 과정에서 Codex는 핵심 협업 도구로 사용됩니다. Codex의 역할과 한계는 책 안의 `Codex 소개와 사용 원칙` 문서에 별도로 정리합니다.

## 현재의 작업 가설

이 저장소는 개인적인 이해를 출발점으로 삼되, 그 이해가 틀릴 수 있음을 전제로 합니다.

현재 가지고 있는 중요한 가설 중 하나는 다음과 같습니다.

딥러닝으로 넘어오면서 AI 연구에는 불확정성, 확률적 동작, 가중치 기반 표현이 더 강하게 반영되었고, 이러한 접근이 GPU 기반 병렬 처리와 결합하면서 빠르게 발전했다는 이해입니다. 과거의 AI가 비교적 명시적이고 논리적인 규칙 중심으로 느껴졌다면, 현대 AI는 대량의 데이터와 가중치, 확률적 추론, 최적화에 기반한 방식으로 발전했다고 보고 있습니다.

이 해석은 개인적인 출발점일 뿐입니다. 표준적인 설명과 다를 수 있고, 일부는 부정확할 수 있습니다. 이 책의 목적은 그런 차이를 숨기는 것이 아니라, 학습과 검증을 통해 무엇이 맞고 무엇이 잘못되었는지 찾아가는 것입니다.

## 문서 작성 원칙

- 원고는 `docs/` 아래 Markdown 파일로 작성합니다.
- 목차는 `mkdocs.yml`의 `nav`에서 관리합니다.
- 새 문서를 추가하면 목차 반영 여부를 함께 확인합니다.
- 외부 자료를 참고하거나 인용한 경우 문서 하단에 출처를 남깁니다.
- 차트와 다이어그램은 가능한 한 Mermaid, SVG, Python 등 오픈소스 기반 도구로 작성합니다.
- 정적 사이트에서 렌더링 가능한 형태로 관리합니다.

## 브랜치 운영

- `main`: 배포 브랜치입니다. `main`에 push되면 GitHub Actions를 통해 GitHub Pages 배포가 실행됩니다.
- `dev`: 일반 작성과 편집 작업을 진행하는 브랜치입니다.

일반적인 문서 작성, 구조 변경, 초안 추가는 `dev` 브랜치에서 진행합니다. 검토가 끝난 변경만 `main`으로 반영합니다.

## 로컬 실행

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
mkdocs serve
```

브라우저에서 `http://127.0.0.1:8000`을 열어 확인합니다.

## 빌드 검증

```bash
mkdocs build
```

현재 로컬 가상환경이 깨져 있거나 `mkdocs` 명령이 없을 수 있습니다. 이 경우 가상환경을 다시 만들고 `requirements.txt`를 설치한 뒤 빌드합니다.

## GitHub Pages 배포

배포 페이지: [https://devchan64.github.io/AiBook/](https://devchan64.github.io/AiBook/)

1. GitHub 저장소의 `Settings > Pages`에서 `Build and deployment`의 `Source`를 `GitHub Actions`로 설정합니다.
2. `main` 브랜치에 push하면 `.github/workflows/deploy.yml`이 정적 사이트를 빌드하고 GitHub Pages에 배포합니다.
