  - “데이터에서 규칙을 배운다”를 엄밀한 정의처럼 쓰기보다, 경험 데이터와 성능 개선의 관점으로 일반화해야 한다는 판단 근거로 사용했다.

## 반영 판단

- 이 절은 규칙 기반 접근과 학습 기반 접근의 차이를 입문적으로 설명하는 문서이므로, 출처 문장을 직접 인용하지 않고 일반화된 표현으로 재서술했다.
- Tom Mitchell의 정의는 널리 알려진 고전적 정의이지만, 본문에서는 직접 인용하지 않고 “경험 데이터와 성능 개선” 관점으로만 반영했다.
- scikit-learn은 Part 3에서 실제로 사용할 API와 용어를 확인하는 공식 자료로 적합하다.
- 보강 과정에서 “규칙을 배운다”는 표현이 사람이 읽을 수 있는 `if-then` 규칙 발견으로 오해될 수 있다고 판단했다.
- 따라서 규칙 기반 접근과 학습 기반 접근을 먼저 표로 비교하고, 모델별 학습 결과가 가중치, 분기, 거리 기준, 확률 관계, 내부 표현처럼 다르게 나타난다는 설명을 추가했다.
- 업무 예시는 고객 문의 분류로 보강했다. 이 예시는 모델 예측이 곧 최종 서비스 결정이 아니라 정책, 승인, 사람 검토와 결합될 수 있음을 보여 주기 위한 것이다.
- 규칙 기반 경로와 학습 기반 경로 비교 도식은 자체 제작 Mermaid 도식이다. 특정 외부 그림을 재작성한 것이 아니라, 본문 설명의 흐름을 시각화하기 위해 추가했다.
- Part 3 초심자 기준에 맞춰 과일 분류 예시를 앞쪽에 추가했다. 이 예시는 알고리즘 설명이 아니라 입력, 관찰값, 라벨, 판단 기준의 직관을 먼저 만들기 위한 것이다.
- 사례(example), 특징(feature), 라벨(label)을 별도 표로 다시 소개해 2.1의 지도학습 설명으로 자연스럽게 이어지도록 했다.

## 주의할 표현

- “규칙을 배운다”를 사람이 읽을 수 있는 명시적 규칙 발견으로 단정하지 않는다.
- 모델이 데이터에 숨어 있는 완전한 진리를 찾는다고 쓰지 않는다.
- 학습 데이터 성능과 새 데이터 성능을 구분한다.
- 지도학습, 비지도학습, 강화학습의 상세 구분은 다음 Chapter로 넘긴다.
- 결정트리는 비교적 규칙처럼 읽을 수 있지만, 모든 모델을 결정트리처럼 설명하지 않는다.
- 모델 출력과 서비스의 최종 정책 결정을 같은 것으로 쓰지 않는다.
