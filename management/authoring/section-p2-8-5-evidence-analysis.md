## 작성 판단

- P2-8.5는 Python 문법 복습의 마지막 절이므로 함수의 고급 기능보다 “작은 재사용”에 집중했다.
- 사용자는 수학과 SW가 연결되는 설명을 중요하게 본다. 따라서 수학의 함수와 Python 함수의 공통점과 차이를 별도 표로 정리했다.
- P2-8.1의 값·타입, P2-8.2부터 P2-8.4까지의 리스트·딕셔너리·반복을 이어 받아, 데이터 한 건을 처리하는 함수와 반복문에서 재사용하는 예시를 중심으로 구성했다.
- `print()`와 `return` 혼동은 초심자에게 자주 생길 수 있으므로 별도 설명했다.
- 기본값(default value)은 라이브러리 API 문서를 읽는 데 자주 필요하므로 입문 수준으로만 포함했다.
- Python 함수 문법을 근거로 삼되, 본문에서는 입력, 처리, 출력의 경계를 정하는 일반 재사용 단위로 정리했다.
- 일반적인 함수 개념과 Python의 `def`, parameter, argument, return 정의를 구분해 표기했다.
- 사용자는 Python이 기존 언어와 다른 함수 사용 방법이 있는지, 클래스 개념에도 일부 차이가 있는지 궁금해했다. 본문에서는 Python 함수가 객체처럼 변수에 담기고 다른 함수에 인자로 전달될 수 있다는 점을 추가했다.
- 클래스는 P2-8.5의 중심 범위가 아니므로 상세 문법은 제외했다. 대신 함수(function), 메서드(method), 클래스(class)를 입문 수준에서 구분하고, `function(value)`와 `value.method()`의 호출 중심이 다르다는 정도만 설명했다.

## 본문에 반영한 안전한 설명

- 함수는 입력을 받아 처리하고 결과를 돌려주는 이름 붙은 코드로 설명할 수 있다.
- 매개변수는 함수를 정의할 때 쓰는 이름이고, 인자는 호출할 때 넣는 실제 값이다.
- `return`은 다음 계산에 쓸 결과를 돌려주고, `print()`는 사람에게 보여주는 출력이다.
- 수학의 함수는 입력과 출력의 관계에 초점을 두고, Python 함수는 실행되는 코드와 재사용에 초점을 둔다.
- 작은 함수는 반복되는 데이터 처리의 입력, 처리, 출력 경계를 분명하게 만든다.
- Python 함수는 객체이므로 변수에 담거나 다른 함수에 인자로 넘길 수 있다.
- 메서드는 객체에 붙어 호출되는 함수 형태로 입문 수준에서 설명할 수 있다.
- 클래스는 객체를 만들기 위한 정의로 간단히 소개하되, 상세 문법은 이후 절로 미룬다.
