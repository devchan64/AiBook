## 사용한 근거

### scikit-learn, Choosing the right estimator

- URL: https://scikit-learn.org/stable/machine_learning_map.html
- 로컬 확인 파일: `.tmp/section-p3-3-2-evidence/scikit-learn-choosing-estimator.html`
- 확인 날짜: 2026-06-25
- 핵심 확인:
  - scikit-learn은 머신러닝 문제에서 적절한 추정기(estimator)를 찾는 일이 어려울 수 있다고 설명한다.
  - 데이터와 문제 유형에 따라 서로 다른 추정기가 더 적합하다고 설명한다.
  - 공식 선택 도식은 범주 예측, 수량 예측, 라벨 유무, 샘플 수, 텍스트 여부 같은 조건으로 후보를 좁히는 흐름을 제공한다.
- 본문 반영:
  - 문제 유형과 제약 조건으로 모델 후보를 좁힌다는 설명의 근거로 사용했다.
  - 단일 모델을 정답처럼 제시하지 않고 “먼저 시도할 후보”로 설명했다.

### scikit-learn, Cross-validation: evaluating estimator performance

- URL: https://scikit-learn.org/stable/modules/cross_validation.html
- 로컬 확인 파일: `.tmp/section-p3-3-2-evidence/scikit-learn-cross-validation.html`
- 확인 날짜: 2026-06-25
- 핵심 확인:
  - 같은 데이터로 학습과 테스트를 모두 하면 방법론적 오류가 될 수 있고, 보지 않은 데이터에서 유용하게 예측하지 못할 수 있다고 설명한다.
  - 지도학습 실험에서는 사용 가능한 데이터의 일부를 테스트 세트로 따로 두는 관행을 설명한다.
  - 상업적 상황에서도 머신러닝은 보통 실험적으로 시작한다는 설명이 있다.
- 본문 반영:
  - 휴리스틱을 최종 결론이 아니라 검증 가능한 출발점으로 다루어야 한다는 설명의 근거로 사용했다.
  - 검증 세부 절차는 P3-4로 넘기고, 이 절에서는 검증의 필요성만 설명했다.

  - 본문에서 특정 수식이나 세부 주장을 직접 인용하지 않았다.

## 일반화한 문장

- “모델 선택은 모델 이름을 고르는 일이 아니라 문제 유형, 데이터 상태, 설명 가능성, 비용, 평가 기준을 함께 보고 후보를 좁히는 일이다.”
- “휴리스틱은 최종 모델을 결정하는 규칙이 아니라 먼저 시도할 후보를 정하는 작업 가설이다.”
- “기준 모델은 최고의 모델이 아니라 복잡한 모델을 비교하기 위한 최소 기준이다.”
