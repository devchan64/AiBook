# P2-7.2 근거 검토: 터미널, 셸, 작업 폴더

확인 날짜: 2026-06-24

## 사용한 주요 자료

| 자료 | 확인 내용 | 반영 위치 |
| --- | --- | --- |
| Free Software Foundation, `Bash Reference Manual` | 셸은 명령 해석기이자 프로그래밍 언어이며, 대화형 또는 비대화형으로 명령을 실행한다는 설명 | 터미널과 셸 구분 |
| Microsoft, `Get-Location` | PowerShell에서 현재 위치를 표시하는 명령 | 작업 폴더 확인 명령 |
| Microsoft, `Set-Location` | PowerShell에서 현재 작업 위치를 지정하는 명령 | 작업 폴더 이동 명령 |
| Python Software Foundation, `os.getcwd` | 현재 작업 디렉터리를 문자열로 반환하는 Python 표준 라이브러리 함수 | current working directory 용어 확인 |

## 작성 판단

- 이 절은 터미널 사용법 전체가 아니라 실행 환경 학습에 필요한 최소 개념만 다루도록 작성했다.
- P2-7.1에서 이미 터미널을 짧게 소개했으므로, P2-7.2에서는 터미널과 셸을 구분하고 작업 폴더가 명령 실행의 기준이 된다는 점을 중심으로 정리했다.
- Python 스크립트 실행의 구체적 흐름은 P2-7.3으로 넘겼다.
- 가상환경 활성화, 패키지 설치, 의존성 관리는 P2-7.4와 P2-7.5의 도메인으로 남겼다.
- Windows/macOS/Linux 명령 차이를 모두 표로 늘리면 절의 중심이 흐려질 수 있으므로, macOS/Linux 계열 명령과 PowerShell 공식 명령을 최소 비교로만 넣었다.
- Colab의 `!` 명령은 P2-3.4와 이어지지만, 이 절에서는 로컬 터미널과 Colab 런타임의 기준 위치가 다르다는 점만 설명했다.

## 본문에 반영한 안전한 설명

- 터미널은 명령을 입력하고 결과를 보는 화면이다.
- 셸은 사용자의 명령을 해석하고 실행하는 프로그램이다.
- 작업 폴더는 현재 명령이 기준으로 삼는 폴더다.
- 상대 경로는 현재 작업 폴더에 따라 의미가 달라진다.
- 파일을 찾을 수 없는 오류는 코드 문제가 아니라 위치 문제일 수 있다.
- Colab 코드 셀의 `!` 명령은 로컬 PC가 아니라 Colab 런타임에서 실행된다.

## 제외한 내용

- 셸 스크립트 작성법
- 파이프, 리다이렉션, 표준 입력/출력
- 권한과 실행 비트
- PATH 환경 변수의 상세 동작
- PowerShell 별칭과 프로파일 설정
- Windows, macOS, Linux별 터미널 설치와 설정

이 내용은 P2-7.3 이후 실습에서 필요할 때 좁게 다시 다루는 편이 안전하다.
