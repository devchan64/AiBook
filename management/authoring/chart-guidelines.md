# 차트와 다이어그램 작성 기준

이 문서는 책 본문에 들어가는 차트와 다이어그램의 작성 기준을 정리합니다. 특히 수학 개념, 모델 계산, 벡터·행렬, 변화율처럼 시각적 설명이 필요한 도식을 만들 때 참고합니다.

## 기본 원칙

- 차트와 다이어그램은 장식보다 설명 목적을 우선합니다.
- 차트는 본문에서 설명하는 하나의 중심 개념을 시각적으로 보강해야 합니다.
- 한 차트에 너무 많은 개념을 넣지 않습니다. 두 개 이상의 중심 개념이 섞이면 차트를 나눕니다.
- 외부 차트를 그대로 복사하지 않습니다. 필요한 경우 공개 라이선스와 인용 가능성을 확인하고, 가능하면 자체 도식으로 다시 작성합니다.
- 외부 자료의 도식이나 차트를 참고해 재작성한 경우에도 근거 메모에 출처와 재작성 여부를 남깁니다.

## 파일 형식과 위치

- 수학 개념, 모델 계산, 벡터·행렬, 변화율 같은 설명 차트는 가능하면 SVG로 작성합니다.
- 복잡한 그림은 `docs/assets/` 아래에 SVG 또는 PNG로 저장합니다.
- SVG 파일은 해당 본문 위치에 맞춰 `docs/assets/part-XX/chapter-YY/` 아래에 둡니다.
- 가능하면 생성 원본도 함께 보관합니다. 수동 SVG라면 SVG 자체가 원본 역할을 합니다.

## SVG 기본 구조

- SVG에는 반드시 `<title>`과 `<desc>`를 넣어 접근성과 검색 가능성을 확보합니다.
- 단일 개념 차트는 기본 `viewBox="0 0 760 430"`을 우선 사용합니다.
- 좌우 비교, 전후 비교, 변환 전후처럼 두 패널이 필요한 차트는 `viewBox="0 0 920 430"`을 사용할 수 있습니다.
- 배경은 단순하게 유지합니다.
- 의미 없는 그라데이션, 불필요한 그림자 효과, 장식용 도형을 넣지 않습니다.

## 수학 차트의 구성 요소

수학 차트에는 가능하면 다음 요소를 일관되게 둡니다.

- 제목 또는 핵심 메시지
- 축(axis) 또는 기준선
- 격자(grid) 또는 눈금(tick)
- 주요 점(point), 선(line), 벡터(vector), 구간(interval)
- 변화량 표시: `Δx`, `Δy`, 방향 화살표 등
- 차트 아래 한 문장 요약

모든 요소가 항상 필요하지는 않습니다. 다만 독자가 차트만 보고도 “무엇을 보라는 것인가”를 파악할 수 있어야 합니다.

## 색상 기준

차트의 색상 의미는 일관되게 사용합니다.

| 역할 | 색상 방향 |
| --- | --- |
| 기본 함수, 주 곡선, 주요 벡터 | 파란색 계열 |
| 변화량, 증가, 비교 강조 | 빨간색 계열 |
| 입력 변화, 기준 이동, 보조 방향 | 초록색 계열 |
| 축, 보조선, 흐름 화살표 | 회색 계열 |

색상만으로 의미를 전달하지 않습니다. 라벨, 선 종류, 위치 설명을 함께 둡니다.

## 글자와 라벨

- SVG 안의 텍스트는 모바일에서도 읽히도록 너무 작게 만들지 않습니다.
- 일반 라벨은 14px 이상, 제목은 18px 이상을 기준으로 합니다.
- 차트 내부의 보이는 문구는 영어를 기본으로 작성합니다.
- 한국어 설명은 본문에서 차트 앞뒤 문장, 이미지 대체 텍스트, `<desc>`, 근거 메모로 보조합니다.
- 수학 기호, 함수식, 표준 용어는 영어 또는 기호를 함께 사용합니다.
- 본문에서 이미 설명한 용어라도 차트 안에서는 짧은 라벨을 붙입니다.

## 본문 삽입 기준

- 본문에서 차트를 삽입할 때는 이미지 바로 앞뒤에 차트가 무엇을 보여 주는지 짧게 설명합니다.
- 차트는 본문을 대체하지 않습니다. 본문 설명을 보강하는 역할로 사용합니다.
- 차트가 본문 주장에 근거로 쓰이면 근거 메모에 차트의 목적, 자체 제작 여부, 외부 자료 참조 여부를 남깁니다.

## 검증

- 새 차트를 추가한 뒤에는 `mkdocs build`로 링크와 렌더링 오류가 없는지 확인합니다.
- 모바일 화면에서 라벨이 너무 작거나 잘리지 않는지 확인합니다.
- 수식, 축, 화살표가 본문 설명과 모순되지 않는지 확인합니다.
