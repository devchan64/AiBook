# 차트와 다이어그램 작성 기준

이 문서는 책 본문에 들어가는 차트와 다이어그램의 작성 기준을 정리합니다. 특히 수학 개념, 모델 계산, 벡터·행렬, 변화율처럼 시각적 설명이 필요한 도식을 만들 때 참고합니다.

## 기본 원칙

- 차트와 다이어그램은 장식보다 설명 목적을 우선합니다.
- 차트는 본문에서 설명하는 하나의 중심 개념을 시각적으로 보강해야 합니다.
- 한 차트에 너무 많은 개념을 넣지 않습니다. 두 개 이상의 중심 개념이 섞이면 차트를 나눕니다.
- 외부 차트를 그대로 복사하지 않습니다. 필요한 경우 공개 라이선스와 인용 가능성을 확인하고, 가능하면 자체 도식으로 다시 작성합니다.
- 외부 자료의 도식이나 차트를 참고해 재작성한 경우에도 근거 메모에 출처와 재작성 여부를 남깁니다.

## 파일 형식과 위치

- 수학 개념, 모델 계산, 벡터·행렬, 변화율 같은 설명 차트는 가능하면 SVG로 작성합니다.
- 복잡한 그림은 `docs/assets/` 아래에 SVG 또는 PNG로 저장합니다.
- SVG 파일은 해당 본문 위치에 맞춰 `docs/assets/part-XX/chapter-YY/` 아래에 둡니다.
- 가능하면 생성 원본도 함께 보관합니다. 수동 SVG라면 SVG 자체가 원본 역할을 합니다.

## SVG 기본 구조

- SVG에는 반드시 `<title>`과 `<desc>`를 넣어 접근성과 검색 가능성을 확보합니다.
- 단일 개념 차트는 기본 `viewBox="0 0 760 430"`을 우선 사용합니다.
- 좌우 비교, 전후 비교, 변환 전후처럼 두 패널이 필요한 차트는 `viewBox="0 0 920 430"`을 사용할 수 있습니다.
- 흐름 화살표, 반복 루프, 하단 요약 박스가 함께 들어가면 `viewBox` 높이를 늘려 충분한 여백을 둡니다.
- 배경은 단순하게 유지합니다.
- 의미 없는 그라데이션, 불필요한 그림자 효과, 장식용 도형을 넣지 않습니다.

## 수학 차트의 구성 요소

수학 차트에는 가능하면 다음 요소를 일관되게 둡니다.

- 제목 또는 핵심 메시지
- 축(axis) 또는 기준선
- 격자(grid) 또는 눈금(tick)
- 주요 점(point), 선(line), 벡터(vector), 구간(interval)
- 변화량 표시: `Δx`, `Δy`, 방향 화살표 등
- 차트 아래 한 문장 요약

모든 요소가 항상 필요하지는 않습니다. 다만 독자가 차트만 보고도 “무엇을 보라는 것인가”를 파악할 수 있어야 합니다.

## 색상 기준

차트의 색상 의미는 일관되게 사용합니다.

| 역할 | 색상 방향 |
| --- | --- |
| 기본 함수, 주 곡선, 주요 벡터 | 파란색 계열 |
| 변화량, 증가, 비교 강조 | 빨간색 계열 |
| 입력 변화, 기준 이동, 보조 방향 | 초록색 계열 |
| 축, 보조선, 흐름 화살표 | 회색 계열 |
| 반복, 되돌아가기, 업데이트 루프 | 파란색 계열 점선 |

색상만으로 의미를 전달하지 않습니다. 라벨, 선 종류, 위치 설명을 함께 둡니다.

- 그래프의 곡선과 동작 화살표가 한 차트에 함께 있으면 같은 색상 계열을 쓰지 않습니다.
- 그래프 곡선은 값이나 함수의 형태를 보여 주고, 화살표는 이동, 반복, 흐름을 보여 주므로 시각 역할을 분리합니다.
- 이 경우 곡선은 청록색, 짙은 중립색, 보라색 등으로 분리하고, 업데이트나 반복 화살표는 파란색 또는 회색 계열로 유지할 수 있습니다.
- 단일 차트 안에서 같은 색은 같은 의미를 가져야 합니다. 같은 파란색이 곡선과 이동 화살표를 동시에 뜻하게 만들지 않습니다.

## 화살표와 흐름 표시

- 차트의 핵심 흐름은 화살표만 보고도 방향을 알 수 있어야 합니다.
- 주요 흐름 화살표는 배경, 카드, 요약 박스와 겹치지 않게 배치합니다.
- 반복 루프나 되돌아가는 화살표는 일반 흐름 화살표보다 굵게 표시하고, 점선이나 색상으로 구분합니다.
- 반복 루프는 하단 요약 문구와 겹치기 쉽기 때문에 요약 박스를 아래로 분리하거나 루프를 별도 여백에 둡니다.
- 화살표 머리는 모바일에서도 보이도록 충분히 크게 둡니다.
- 화살표 머리는 선 두께와 균형을 맞춥니다. 머리만 과하게 크면 흐름보다 장식처럼 보이므로 마커 크기를 줄이거나 선 두께를 함께 조정합니다.
- 화살표가 선, 카드 테두리, 텍스트와 가까우면 의미가 약해지므로 주변에 여백을 둡니다.
- 화살표 머리는 내용 박스 안쪽으로 파고들지 않게 합니다. 박스를 가리키는 화살표는 끝점을 박스 테두리 바깥이나 테두리 직전으로 두고, 머리가 텍스트 영역을 덮지 않는지 확인합니다.
- 화살표가 카드 사이의 관계를 보여 줄 때는 카드 내부 텍스트보다 카드의 바깥 경계나 빈 여백을 향하게 합니다.
- 흐름을 설명하는 라벨은 화살표 바로 위나 아래에 둘 수 있지만, 선과 겹치면 안 됩니다.
- 화살표 라벨이 선과 가까우면 흰색 또는 배경색 캡슐 박스 안에 넣어 독립된 읽기 영역으로 만듭니다.
- 화살표가 굵어질수록 라벨은 더 멀리 두거나 별도 박스로 분리합니다.
- 되돌아가는 루프는 본문 요약 박스 위를 통과하지 않게 하고, 요약 박스와 최소한 한 줄 높이 이상의 간격을 둡니다.

## 겹침 방지 기준

- 텍스트는 선, 화살표, 카드 테두리, 아이콘, 데이터 점과 겹치면 안 됩니다.
- 카드 내부 텍스트는 카드 테두리와 충분한 안쪽 여백을 둡니다.
- 차트 하단 요약 문구는 도식의 일부가 아니라 해석 문장입니다. 흐름선, 루프, 화살표 머리와 분리합니다.
- 강조 박스, 라벨 박스, 요약 박스가 서로 붙어 있으면 독자는 하나의 요소로 오해할 수 있으므로 시각적 간격을 둡니다.
- 텍스트가 선 위에 올라가면 선을 옮기는 것을 우선하고, 어려우면 텍스트에 배경 박스를 둡니다.
- 한 줄 라벨이 길어져 화살표나 도형과 충돌하면 문구를 줄이거나 두 줄로 나눕니다.
- 차트 안에서 가장 중요한 흐름은 가장 먼저 보이도록 하고, 보조 설명은 별도 박스나 본문 설명으로 넘깁니다.

## 글자와 라벨

- SVG 안의 텍스트는 모바일에서도 읽히도록 너무 작게 만들지 않습니다.
- 일반 라벨은 14px 이상, 제목은 18px 이상을 기준으로 합니다.
- 화살표 라벨, 요약 문구, 카드 내부 설명이 서로 겹치지 않도록 세로 여백을 먼저 확보합니다.
- 한 도표 안에서는 텍스트 정렬 규칙을 먼저 정하고 일관되게 적용합니다.
- 패널 제목, 설명 문장, 긴 문구는 좌측 정렬을 기본으로 합니다.
- 작은 카드, 원, 버튼형 박스처럼 짧은 상태값이나 후보명을 담는 요소는 가운데 정렬을 사용할 수 있습니다.
- 가운데 정렬을 사용할 때는 눈대중으로 `x` 좌표를 보정하지 말고, 요소의 중앙 좌표와 `text-anchor="middle"`을 사용합니다.
- 같은 성격의 요소는 같은 정렬 방식을 사용합니다. 예를 들어 후보 카드가 가운데 정렬이면 선택 결과 카드와 거절 카드도 가운데 정렬합니다.
- 좌측 정렬 요소와 가운데 정렬 요소가 한 차트에 함께 있을 수는 있지만, 역할이 달라야 합니다. 설명문은 좌측 정렬, 짧은 라벨은 가운데 정렬처럼 구분합니다.
- 차트 내부의 보이는 문구는 영어를 기본으로 작성합니다.
- 한국어 설명은 본문에서 차트 앞뒤 문장, 이미지 대체 텍스트, `<desc>`, 근거 메모로 보조합니다.
- 수학 기호, 함수식, 표준 용어는 영어 또는 기호를 함께 사용합니다.
- 본문에서 이미 설명한 용어라도 차트 안에서는 짧은 라벨을 붙입니다.

## 본문 삽입 기준

- 본문에서 차트를 삽입할 때는 이미지 바로 앞뒤에 차트가 무엇을 보여 주는지 짧게 설명합니다.
- 차트는 본문을 대체하지 않습니다. 본문 설명을 보강하는 역할로 사용합니다.
- 차트가 본문 주장에 근거로 쓰이면 근거 메모에 차트의 목적, 자체 제작 여부, 외부 자료 참조 여부를 남깁니다.

## 검증

- 새 차트를 추가한 뒤에는 `mkdocs build`로 링크와 렌더링 오류가 없는지 확인합니다.
- 모바일 화면에서 라벨이 너무 작거나 잘리지 않는지 확인합니다.
- 수식, 축, 화살표가 본문 설명과 모순되지 않는지 확인합니다.
- SVG를 직접 브라우저에서 열어 화살표, 라벨, 요약 박스가 겹치지 않는지 확인합니다.
- MkDocs 페이지에 삽입된 상태에서도 확인합니다. 단독 SVG와 본문 삽입 상태의 표시 크기가 다를 수 있습니다.
- 차트를 수정한 뒤에는 다음 항목을 눈으로 확인합니다.
  - 화살표 방향이 명확한가
  - 화살표 머리가 보이는가
  - 라벨이 선이나 화살표와 겹치지 않는가
  - 하단 요약 문구가 도식 요소와 분리되어 있는가
  - 모바일 폭에서도 텍스트가 잘리거나 겹치지 않는가
