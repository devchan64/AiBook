## 작성 판단

- 사용자는 전통적인 자료구조 개론을 일부 소개할 필요가 있다고 판단했다.
- P2-9.1은 자료구조의 필요성을 설명하는 도입 절이므로, 배열, 연결 리스트, 스택, 큐, 트리, 그래프, 해시 테이블 설명을 모두 9.1에 길게 넣으면 절의 중심 질문이 흐려진다.
- 이에 따라 P2-9.4를 보충학습으로 추가해 전통 자료구조 이름을 Chapter 9 후반부에서 별도로 천천히 읽도록 구성했다.
- 이 절의 목표는 자료구조 구현이 아니라, 자료구조 이름이 어떤 데이터 처리 질문을 대표하는지 설명하는 것이다.
- Python의 list와 dictionary를 전통 자료구조와 1:1로 동일시하지 않고, 사용 관점에서 빌려 쓸 수 있는 감각으로 설명했다.
- 보강 요청에 따라 전통 자료구조를 왜 뒤에서 다시 보는지, 자료구조를 고를 때 어떤 질문을 해야 하는지, 같은 데이터를 여러 구조로 표현하는 예시, 오해하기 쉬운 지점을 추가했다.
- AI 실습과의 연결을 강화하기 위해 토큰 시퀀스, 라벨 맵, 문서 관계, 검색·추천 구조, 작업 큐를 예시로 보강했다.
- 사용자가 각 자료구조를 돌려볼 수 있는 Python 예제를 요청했다. 이에 따라 배열 감각, 연결 리스트 감각, 스택, 큐, 트리, 그래프, 키 기반 검색을 각각 짧은 실행 예제로 추가했다.

## 본문에 반영한 안전한 설명

- 자료구조 이름은 외울 목록이 아니라 문제 해결 질문을 대표한다.
- 배열은 위치와 순서, 연결 리스트는 다음 항목을 가리키는 발상, 스택과 큐는 넣고 꺼내는 규칙을 보여 준다.
- 트리는 계층 구조, 그래프는 관계 구조를 표현하는 데 유용하다고 설명한다.
- 딕셔너리와 해시 테이블은 키 기반 검색과 연결되지만, Python 딕셔너리와 해시 테이블 구현을 완전히 같은 수준의 설명으로 단정하지 않는다.
- AI 실습에서 리스트, 매핑, 집합, 표, 그래프, 큐 감각이 반복적으로 등장한다고 연결했다.
- 자료구조 선택은 순서, 키 기반 검색, 관계 이동이라는 세 질문으로 입문 수준에서 설명할 수 있다.
- Python 리스트와 전통적인 배열, Python 딕셔너리와 해시 테이블, 트리와 그래프의 관계를 오해하지 않도록 사용 관점과 구현 관점을 구분했다.
- 자료구조는 성능만이 아니라 코드의 의도와 데이터 의미를 드러내는 역할도 한다고 설명했다.
- 실행 예제는 완전한 구현 수업이 아니라, 각 자료구조의 동작 감각을 확인하는 최소 코드로 제한했다.
