## 작성 판단

- P2-11.1은 배열 생성과 shape 확인을 다루므로, P2-11.2는 배열에서 값을 읽고 구간을 자르고 축 방향으로 요약하는 최소 감각에 집중했다.
- 고급 인덱싱, 불리언 마스크, view/copy는 중요한 주제지만 초심자에게 과도하므로 제외했다.
- `axis=0`을 “0번 행”으로 오해하지 않도록, 인덱싱의 숫자와 axis 인자의 숫자를 분리해 설명했다.
- 도식 `index-slice-axis-map.svg`는 위치 선택, 행 선택, 열 선택, 축 방향을 한 배열 위에서 보여 주기 위해 자체 제작했다.
- 도식 `axis-reduction.svg`는 `sum(axis=0)`과 `sum(axis=1)`의 결과 shape 차이를 보여 주기 위해 자체 제작했다.
- 사용자의 “예제와 도식 보강” 요청에 따라 `slice-start-stop-step.svg`, `dataset-row-column-selection.svg`를 추가했다. 두 도식은 자체 제작이며 외부 도식을 복제하지 않았다.
- 예제 코드는 문서의 설명 순서와 동일하게 인덱싱, 슬라이싱, 부분 배열, 축 요약을 출력하도록 작성했고, 이후 1차원 `start:stop:step` 슬라이싱과 데이터셋형 행/열 선택 예제를 추가했다.

## 본문에 반영한 안전한 설명

- 인덱싱은 하나의 위치를 고르는 일로 설명했다.
- 슬라이싱은 구간을 남기는 일로 설명했다.
- 2차원 배열에서는 `data[행, 열]`로 읽는다고 설명했다.
- `:`는 해당 차원의 전체를 선택하는 문법으로 설명했다.
- axis는 배열의 차원이며, 2차원 배열에서 axis 0은 행 방향, axis 1은 열 방향으로 설명했다.
- `sum(axis=0)`과 `sum(axis=1)`은 위치 선택이 아니라 요약 계산의 방향 선택이라고 설명했다.
- `start:stop:step`에서 stop 위치는 선택되지 않는 경계임을 도식과 예제로 보강했다.
- 행은 샘플, 열은 특징으로 읽는 데이터셋 관점을 별도 도식과 평균 계산 예제로 보강했다.
