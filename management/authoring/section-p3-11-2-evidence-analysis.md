# P3-11.2 결정 경계(decision boundary) 근거 검토

## 핵심 주장

### 1. 결정 경계는 분류 모델이 class를 나누는 기준이다

- 근거:
  - scikit-learn, `1.1.11. Logistic regression`
  - 로지스틱 회귀의 선형 결정 함수 설명
- 반영 방식:
  - 출력 점수 관점에서 입력 공간 관점으로 옮겨 설명
  - `class를 가르는 기준선 또는 기준면`으로 일반화

### 2. 로지스틱 회귀에서 threshold 0.5는 선형 점수 0과 연결된다

- 근거:
  - scikit-learn, `LogisticRegression`
  - sigmoid와 decision function의 일반 설명
- 반영 방식:
  - 엄밀한 식 유도는 생략
  - `sigmoid output 0.5 <-> linear score 0`을 입문 수준으로 설명

### 3. 1차원에서는 경계가 점처럼, 2차원에서는 선처럼 보인다

- 근거:
  - 선형 분류기의 입력 차원별 기하학적 일반 설명
- 반영 방식:
  - 1D cutoff point, 2D separating line으로 단순화
  - 고차원 초평면은 언급만 하고 깊게 다루지 않음

### 4. coefficient와 intercept는 경계의 방향과 위치에 관여한다

- 근거:
  - scikit-learn, `1.1.11. Logistic regression`
- 반영 방식:
  - 계수는 방향, 절편은 평행 이동 감각으로 소개
  - 엄밀한 기울기 유도는 범위 밖으로 둠

### 5. threshold 변화는 경계 해석과 class 영역을 바꿀 수 있다

- 근거:
  - 로지스틱 회귀의 `predict_proba`, `decision_function`, `predict` 관계
- 반영 방식:
  - 별도 Python 예제로 threshold 0.5와 0.7 비교
  - 운영 정책과 경계 해석을 연결

### 6. 역사 설명은 `분류를 공간 분리 문제로 읽는 관점`이 왜 중요한지 보여 주는 수준에서 정리한다

- 근거:
  - 선형 분류기와 decision boundary에 대한 일반적 머신러닝 입문 설명
- 반영 방식:
  - 회귀의 값 추정 관점
  - 분류의 공간 분리 관점
  - 같은 로지스틱 회귀를 점수 모델과 경계 모델 두 관점으로 읽는 흐름
  - 뒤의 SVM, 트리, 신경망으로 이어지는 연결만 남기고 세부 연표는 생략

## 본문 반영 판단

- 11.2는 11.1의 확률 출력 설명을 반복하지 않고, 입력 공간에서의 분리 관점으로 이동한다.
- Python 예제는 2차원 toy data를 사용해 계수, intercept, decision score, predict_proba, prediction을 함께 출력한다.
- 도식은 경계의 기하를 완전한 plot 대신 구조적 Mermaid 흐름으로 설명한다.
- 도식은 각각 다른 질문을 맡는다: `1D 경계는 어디인가`, `2D에서는 선을 어떻게 읽는가`, `threshold 변화가 영역을 어떻게 줄이는가`, `경계 근처 샘플은 왜 애매한가`.
- 선형 경계의 한계는 뒤 알고리즘 절로 넘기되, 왜 다음 모델들이 필요한지 정도는 남긴다.
- 역사 서술은 로지스틱 회귀 자체의 연표보다 `decision boundary 관점의 교육적 의미`를 우선한다.
- 사례는 합격, 이탈, 의료, 대출, 스팸처럼 `여러 특징의 조합이 판단을 만든다`는 점을 보여 주는 방향으로 보강한다.
- 주요 논의점은 경계의 기하학 자체보다 `경계 근처 샘플`, `데이터 변화`, `서비스 정책`처럼 초심자가 실무와 연결해 오해하기 쉬운 지점을 우선한다.
