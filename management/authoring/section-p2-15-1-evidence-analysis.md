# P2-15.1 근거 검토: 수식을 코드로 옮기는 작은 절차

## 사용한 주요 자료

| 자료 | 확인한 핵심 | 본문 반영 지점 |
| --- | --- | --- |
| Python documentation, `An Informal Introduction to Python` | Python은 계산기처럼 산술 표현식을 실행할 수 있고, 변수 할당과 기본 연산을 제공한다. | 작은 반복문과 산술 계산 예시 |
| NumPy documentation, `NumPy: the absolute basics for beginners` | NumPy 배열은 같은 종류의 값을 담고 배열 연산을 수행하는 기본 객체로 설명된다. | 반복문 계산을 NumPy 배열 계산으로 줄이는 설명 |
| Matplotlib documentation, `Quick start guide` | `Figure`, `Axes`, `plot` 흐름으로 데이터를 시각화한다. | 실제값과 예측값을 그래프로 확인하는 예시 |

## 범위 판단

- P2-15.1은 Part 2의 수학과 Python 도구를 하나의 작은 절차로 묶는 절이다.
- MSE를 예시로 사용하되, 회귀 모델이나 최적화 알고리즘으로 확장하지 않았다.
- 수식 증명보다 기호를 코드 변수와 반복 계산으로 바꾸는 절차에 집중했다.

## 조심한 지점

- NumPy 코드가 짧다는 이유로 처음부터 더 쉽다고 단정하지 않았다.
- 반복문으로 의미를 확인한 뒤 NumPy로 줄이는 흐름을 제안했다.
- MSE를 평가 지표 전체 설명으로 확장하지 않고, 수식-코드 변환 예제로만 사용했다.

## 이 절의 핵심 정리

- 중심 질문은 “수식을 어떻게 작은 코드 절차로 옮기는가?”다.
- 핵심 답은 기호를 확인하고, 데이터 모양을 구분하고, 반복 계산 또는 배열 계산으로 옮긴 뒤 중간값과 그래프로 확인하는 것이다.
