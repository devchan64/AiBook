## 작성 판단

- 이 절은 프로그래밍 기초 지식이 없는 독자를 기준으로 작성했다.
- P2-3.4의 Colab/로컬 실행 안내와 일부 내용이 중복되지만, P2-7.1에서는 실행 환경의 기본 용어를 더 낮은 수준에서 다시 설명했다.
- 운영체제별 설치 절차, IDE 설정, PATH 문제, PowerShell 실행 정책은 깊게 다루지 않았다. 초심자에게 필요한 구분은 `터미널 명령`, `Python 코드`, `코드 셀 명령`, `패키지 설치`, `import`의 차이라고 판단했다.
- 사용자의 피드백에 따라 Chapter 7은 간단한 2개 절이 아니라 여러 절로 나누는 방향으로 조정했다. P2-7.1은 큰 그림을 잡고, 터미널과 작업 폴더는 P2-7.2, Python 인터프리터와 스크립트는 P2-7.3, 가상환경과 패키지는 P2-7.4, 의존성과 재현성은 P2-7.5에서 다루는 구조로 목차를 확장했다.
- Python 설치는 Chapter 7 앞부분에서 다루지 않고 P2-7.6 보충수업으로 넘기기로 했다. 프로그래밍 기초가 없는 독자에게 설치 절차가 먼저 나오면 실행 환경 개념보다 운영체제별 설치 문제에 먼저 막힐 수 있기 때문이다. 또한 이 책의 초반 실습은 Colab 같은 브라우저 실행 환경으로도 어느 정도 대체할 수 있으므로, 로컬 설치는 필요성이 분명해지는 후반부 보충수업으로 미루는 편이 안전하다고 판단했다.
- 사용자의 피드백에 따라 Python의 역사를 짧게 언급했다. 다만 이 절의 중심 질문은 언어사가 아니라 실행 환경이므로, Guido van Rossum, ABC 언어의 영향, 1991년 공개, 입문 언어로서의 특성만 공식 FAQ에 근거해 연결했다. AI에서 Python이 자주 보이는 이유는 역사적 사실로 단정하기보다, 읽기 쉬운 고수준 언어, 대화형 실행, 실험 코드, 데이터 처리 도구 생태계가 연결된 설명으로 정리했다.

## 본문에 반영한 안전한 설명

- Python 인터프리터는 Python 코드를 읽고 실행하는 프로그램이다.
- Python은 사람이 읽고 쓰기 쉬운 고수준 언어와 실제 시스템 작업에 쓸 수 있는 실행 도구 사이에서 발전한 언어로 소개할 수 있다.
- AI 학습과 실험의 입구에서 Python을 자주 만나는 이유는 수식·실험·데이터 처리·라이브러리 생태계와 연결해 설명할 수 있다.
- 터미널 명령과 Python 코드는 다르다.
- 로컬 환경은 내 컴퓨터에서 코드가 실행되는 조건이다.
- 실행 환경은 코드가 실제로 실행되는 자리다.
- 가상환경은 프로젝트별 Python 실행 공간으로 이해할 수 있다.
- 패키지 설치와 `import`는 다른 일이다.
- P2-7.1에서는 위 개념을 모두 깊게 설명하지 않고, 왜 이어서 학습해야 하는지의 큰 지도를 제공한다.
