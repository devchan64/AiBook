# Chapter 4 도메인 경계 검토

이 문서는 `docs/chapters/chapter-04/`의 섹션 간 도메인 오염을 검토한 관리 메모입니다.

## 섹션별 책임

| 섹션 | 중심 질문 | 다루는 내용 | 넘기는 내용 |
| --- | --- | --- | --- |
| 4.1 모델이라는 말에 익숙해지기 | 모델이란 말을 어떻게 이해할 것인가? | 모델/모형 비유, 현실의 단순화, 모델의 한계 | 입력/출력/데이터 세부 정의, 평가 지표, 서비스 설계 |
| 4.2 입력, 출력, 데이터 | 모델을 설명할 때 무엇을 넣고 무엇을 얻고 어떤 사례를 볼 것인가? | 입력, 출력, 데이터의 기본 역할과 데이터 모양 | 과제 유형 분류, 평가 지표, 모델 선택 |
| 4.3 특징, 표현, 파라미터 | 입력은 모델 안에서 어떤 계산 재료가 되는가? | 특징, 표현, 파라미터의 위치와 혼동 방지 | 학습 알고리즘, 모델 구조, 인텐트 라우팅, LLM 생성 설정의 세부 동작 |
| 4.4 문제 정의가 모델을 결정하는 방식 | 현실 목표를 어떤 AI 문제로 정의할 것인가? | 모델링 과제, 출력 정의, 데이터 범위, 평가 기준의 연결 | 평가 지표 계산, 최적화, 배포 아키텍처, 운영 모니터링 |

## 조정한 오염 지점

| 지점 | 문제 | 조정 |
| --- | --- | --- |
| 4.2 출력 설명 | 분류, 회귀, 생성, 평가 지표까지 설명해 4.4와 겹침 | 4.2는 출력의 형태와 필요한 데이터 변화만 설명하고, 과제 성격과 평가는 4.4로 넘김 |
| 4.2 데이터셋 설명 | 문제 정의와 평가까지 확장됨 | 데이터의 모양 변화만 남기고, 모델 후보와 평가 기준은 4.4로 분리 |
| 4.3 인텐트 분석 | 파라미터 설명 흐름 앞에 들어와 NLU/서비스 구조로 확장됨 | `용어 경계 메모`로 이동해 인텐트는 모델 파라미터가 아니라 입력 해석 결과임을 짚고 후속 장으로 넘김 |
| 4.3 temperature | LLM 생성 설정이 모델 내부 파라미터 설명과 섞임 | 모델 파라미터와 생성 설정값의 혼동 방지로만 제한 |
| 4.4 서비스 운영 | 라우팅, 권한, 배포, 모니터링까지 설계처럼 읽힐 수 있음 | 4.4 범위 문단을 추가하고 서비스 아키텍처와 프로젝트 장으로 넘김 |

## 유지한 경계 표현

- 4.2는 `입력/출력/데이터`의 기본 역할을 설명합니다.
- 4.3은 `특징/표현/파라미터`의 기본 위치를 설명합니다.
- 4.4는 `현실 목표 -> 모델링 과제 -> 입력/출력/데이터/평가 기준`의 연결을 설명합니다.
- 세부 알고리즘, 평가 지표 계산, 최적화, 배포 운영은 Chapter 4에서 깊게 다루지 않습니다.
