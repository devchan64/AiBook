## 작성 판단

- P2-9.1은 P2-8에서 배운 Python 문법을 자료구조 관점으로 일반화하는 도입 절이다.
- 사용자는 전통적인 자료구조 개론도 일부 소개할 필요가 있다고 판단했다. 이에 따라 배열, 연결 리스트, 스택, 큐, 트리, 그래프, 해시 테이블을 상세 구현이 아니라 입문용 지도 수준으로 추가했다.
- 선형 구조, 비선형 구조, 키 기반 구조의 구분은 엄밀한 학술 분류로 단정하지 않고, 초심자가 자료구조 이름을 읽을 때 방향을 잡기 위한 설명으로 제한했다.
- 사용자는 그래프를 자세히 배운 경험이 부족하다고 했고, 이후 그래프를 다룰 계획이 있다. 따라서 9.1에서는 그래프를 깊게 설명하지 않고 “관계를 표현하는 구조”라는 예고 수준으로만 둔다.
- 자료구조를 성능 최적화만으로 설명하면 초심자에게 부담이 커진다. 본문에서는 검색, 추가, 삭제, 순회 같은 연산과 함께 읽되, 코드 의도와 데이터 의미를 드러내는 역할도 강조했다.
- ADT와 구현의 구분은 이후 stack, queue, tree, graph를 다룰 때 오해를 줄이기 위해 짧게 포함했다.
- Python 예시는 Chapter 8에서 이미 다룬 list, dict, set 감각을 재사용했다.

## 본문에 반영한 안전한 설명

- 자료구조는 데이터를 담는 모양과 그 데이터를 다루는 방법을 함께 생각하는 개념이다.
- 전통적인 자료구조 이름은 외울 목록이 아니라, 데이터를 다루는 문제 유형을 대표하는 이름으로 소개한다.
- 선형 구조는 앞뒤 순서가 중요한 구조, 비선형 구조는 계층이나 관계가 중요한 구조, 키 기반 구조는 키로 값을 찾는 일이 중요한 구조로 입문 수준에서 설명한다.
- 자료구조는 검색, 추가, 삭제, 순회 같은 연산과 연결된다.
- 같은 데이터도 순서대로 처리할지, 이름으로 찾을지, 관계를 표현할지에 따라 다른 구조로 담을 수 있다.
- 추상 자료형은 가능한 값과 연산을 설명하고, 구현은 그것을 실제 코드와 메모리에서 표현하는 방식이다.
- AI 실습에서는 문장 목록, 라벨 맵, 중복 토큰, 표 형식 데이터, 문서 관계처럼 서로 다른 구조를 계속 만난다.
