# 저자 관점 노트

이 문서는 본문에 바로 넣기 전, 나중에 소개나 장별 관점 문장으로 다듬어 사용할 메모를 모읍니다.

## AI 도구 활용 관점

### 원문 메모

AI란 나의 비전문적인 지식 분야를 도와주어 문제를 해결하게 지원해주는 도구로 활용, 반복적인 업무를 자동화하여 다른 업무에 집중할수록 도와주는 도구

### 정제 후보

AI는 비전문 영역의 문제를 이해하고 해결하는 과정을 보조하는 도구로 활용할 수 있다. 또한 반복적인 업무를 자동화해, 사람이 더 중요한 판단과 창의적 업무에 집중하도록 돕는 도구로 볼 수 있다.

### 본문 사용 시 주의

- 이 문장은 AI의 전체 정의가 아니라 활용 관점으로 사용합니다.
- “비전문 영역을 대신 해결한다”가 아니라 “이해와 문제 해결을 보조한다”로 표현합니다.
- 반복 업무 자동화는 장점이지만, 결과 검토와 책임은 사람에게 남는다는 점을 함께 설명합니다.

## 프롬프팅과 알고리즘 이해의 우선순위

### 원문 메모

현재는 알고리즘의 이해보다는 일반적인 표현으로 프롬프팅하는 게 더 생산성에 도움이 된다고 생각하고 있다.

나중에 고민할 내용.

### 정제 후보

AI 도구를 실무적으로 활용하는 단계에서는 내부 알고리즘을 먼저 깊게 이해하는 것보다, 문제를 명확한 자연어로 설명하고 원하는 결과를 검토 가능한 형태로 요청하는 능력이 더 직접적인 생산성으로 이어질 수 있다.

### 본문 사용 시 주의

- 알고리즘 이해가 불필요하다는 의미로 쓰지 않습니다.
- 입문 단계의 활용 관점과 연구·개발 단계의 이해 요구를 구분합니다.
- 프롬프팅은 결과를 대신 보증하지 않으므로, 근거 확인과 오류 검토가 함께 필요하다고 설명합니다.

## 기호 기반 AI와 데이터 라벨링의 관계

### 원문 메모

심볼릭 AI와 데이터 라벨링의 관계가 있을 것 같다는 생각이 든다.

### 정제 후보

데이터 라벨링은 학습 데이터에 사람이 읽을 수 있는 이름이나 범주를 붙이는 작업이다. 이 점에서 라벨은 일종의 기호처럼 작동한다. 다만 데이터 라벨링은 주로 머신러닝 학습을 위한 데이터 준비 과정이고, 기호 기반 AI(symbolic AI)는 명시적 기호와 규칙을 조작해 추론하려는 접근이므로 둘을 같은 것으로 보지는 않는다.

기호 기반 AI와 데이터 라벨링은 “사람이 세계를 구분 가능한 이름과 범주로 정리한다”는 점에서 만난다. 그러나 전자는 규칙과 추론의 구조를 직접 설계하는 데 가깝고, 후자는 모델이 패턴을 학습할 수 있도록 데이터에 의미 표지를 제공하는 데 가깝다.

### 본문 사용 시 주의

- “라벨링은 심볼릭 AI다”라고 단정하지 않습니다.
- 라벨은 기호처럼 쓰일 수 있지만, 라벨링 자체는 머신러닝 파이프라인의 데이터 준비 과정으로 설명합니다.
- 이 내용은 2.1의 핵심 주제라기보다 지도학습, 데이터, 특징, 모델을 설명하는 장에서 다시 다룹니다.
