## 작성 판단

- P2-8.2는 데이터 묶음을 다루는 첫 절이므로 리스트, 딕셔너리, 반복을 하나의 흐름으로 묶었다.
- P2-8.1에서 “값에 붙인 이름”을 설명했으므로, 리스트의 mutable 성질과 같은 리스트를 두 이름이 가리킬 수 있다는 점을 짧게 연결했다.
- 얕은 복사, 깊은 복사, 성능, 제너레이터, 고급 컴프리헨션은 범위를 넘기 때문에 제외했다.
- Python의 특징을 드러내기 위해 `for item in items` 형태를 먼저 설명하고, `range(len(...))`는 위치가 필요할 때의 보조 방식으로 배치했다.
- 사용자는 Python이 이터레이터 방식의 반복 구조를 갖는다고 이해하고 있으며, 다양한 반복 패턴을 경험한다고 했다. 본문에서는 “for가 iterable에서 iterator를 얻어 값을 하나씩 소비하는 구조”라는 입문 수준 표현으로 일반화했다.
- 반복 패턴은 항목 반복, 인덱스 반복, 키-값 반복, 병렬 반복, 필터링 반복으로 나누어 예시를 추가했다.
- 사용자는 딕셔너리에서 범용적인 맵 성능 경험을 갖고 있다고 했다. 본문에서는 성능을 절대화하지 않고, “키로 값을 빠르게 찾는 범용적인 mapping 구조”라는 안전한 표현으로 보강했다.
- 사용자는 예시가 많이 필요하다고 했다. 리스트, 딕셔너리, 반복을 각각 실무·AI 실습 상황으로 읽을 수 있도록 점수 목록, 문장 목록, 모델 출력, 파일 이름, 설정값, 라벨 맵, 단어 카운트, 샘플 ID, 컬럼 설명, 정규화, 누적, 조건 분리, 라벨 분포 예시를 추가했다.
- AI 실습 연결을 위해 학생 점수 예시를 데이터셋(dataset), 샘플(sample), 필드(field) 관점으로 일반화했다.

## 본문에 반영한 안전한 설명

- 리스트는 순서가 있는 값 묶음이다.
- 딕셔너리는 키와 값의 묶음이며, 위치보다 이름표가 중요할 때 어울린다.
- 딕셔너리는 hashable key를 값에 연결하는 mapping 구조이며, 라벨 맵, 설정값, 카운트, 메타데이터처럼 이름으로 값을 찾는 데이터에 강하다.
- Python의 `for` 반복은 묶음의 항목을 하나씩 꺼내 처리하는 방식으로 읽는 것이 자연스럽다.
- 이터러블은 하나씩 꺼낼 수 있는 대상이고, 이터레이터는 실제로 값을 하나씩 내보내는 흐름으로 설명할 수 있다.
- Python 반복 패턴은 항목, 위치, 키-값, 병렬 묶음, 조건 필터링처럼 나뉘어 보일 수 있다.
- 위치가 필요하면 `enumerate()`를 사용할 수 있다.
- 딕셔너리의 키와 값을 함께 보려면 `.items()`를 사용할 수 있다.
- 반복 중 원본 컬렉션을 직접 수정하는 코드는 조심해야 하며, 새 컬렉션을 만드는 방식이 더 명확할 수 있다.
