## 작성 판단

- P2-8.2는 데이터 묶음을 다루는 첫 절이므로 리스트에 집중하도록 분리했다.
- P2-8.1에서 “값에 붙인 이름”을 설명했으므로, 리스트의 mutable 성질과 같은 리스트를 두 이름이 가리킬 수 있다는 점을 짧게 연결했다.
- Python 튜토리얼이 리스트를 스택처럼 쓰는 방식, 큐로 쓸 때의 비효율, `deque`, tuple, set, dictionary를 분리해 설명하므로, 본문에서도 리스트를 다른 자료구조와 역할 기준으로 비교했다.
- 사용자는 예시가 많이 필요하다고 했다. 리스트를 실무·AI 실습 상황으로 읽을 수 있도록 점수 목록, 문장 목록, 모델 출력, 파일 이름 예시를 추가했다.
- 사용자가 리스트 초기화 패턴 예시를 요청했다. 값이 이미 있는 리스트, 빈 리스트 후 `append`, 같은 단순값으로 길이 맞추기, 중첩 리스트 주의를 포함했다.
- 반복으로 새 리스트를 만들거나 리스트 컴프리헨션을 쓰는 예시는 반복 구조가 중심이므로 P2-8.4로 재배치했다.
- Python 리스트를 근거로 삼되, 본문에서는 순서가 있는 데이터 묶음이라는 일반 자료구조 관점으로 정리했다.
- 사용자는 리스트와 배열(array)의 차이를 의식하고 있다. 본문에서는 리스트를 배열과 동일시하지 않고, Python 리스트를 일반 목적 컨테이너로, 수치 계산 배열을 별도 계산 구조로 분리했다.
- Python 공식 문서에서도 `list`와 `array`는 별도 항목으로 설명된다. `array`는 숫자값을 효율적으로 담는 배열이며 type code로 값의 타입이 제한된다는 점을 근거로 삼았다.
- NumPy 문서는 `ndarray`를 homogeneous, N-dimensional array로 설명하고, shape, axis, dtype을 통해 다차원 배열을 다룬다. 본문에서는 중첩 리스트가 행렬처럼 보일 수 있으나 `ndarray`와 같은 계산 구조는 아니라고 구분했다.
- 일반적인 순서 있는 묶음과 Python 리스트 정의를 구분해 표기했다.
- 사용자는 `concat`, `join`, `slice`, `splice` 같은 동작이 Python 리스트에서 축약 문법으로 보인다고 했다. 본문에서는 Python에 JavaScript식 `splice()` 메서드가 없다는 점, `join()`은 문자열(str) 메서드라는 점, 리스트 연결과 슬라이싱은 시퀀스 문법으로 읽어야 한다는 점을 분리했다.

## 본문에 반영한 안전한 설명

- 리스트는 순서가 있는 값 묶음이며, Python 기본 문법 복구에서 먼저 다루는 일반 목적 컨테이너다.
- Python 리스트는 위치로 접근할 수 있지만, 수치 계산용 배열(array)과 같은 말은 아니다.
- 배열(array), 벡터(vector), 행렬(matrix)은 P2-3과 P2-11에서 별도 계산 구조로 다룬다.
- 중첩 리스트는 행렬처럼 보일 수 있지만, shape, axis, dtype을 가진 NumPy `ndarray`와 동일하지 않다.
- 리스트는 모든 묶음을 대신하는 구조가 아니라, 순서가 있고 변경 가능한 일반 목적 시퀀스로 먼저 읽는다.
- 리스트는 점수, 문장, 모델 출력, 파일 이름처럼 같은 목적이나 맥락의 값을 순서대로 담을 때 어울린다.
- 리스트는 이미 아는 값으로 시작하거나, 빈 리스트로 시작하거나, 같은 단순값으로 길이를 맞춰 시작할 수 있다.
- 반복 규칙으로 새 목록을 만드는 방법은 P2-8.4에서 다룬다.
- 리스트는 mutable value이므로 같은 리스트를 여러 이름이 가리킬 수 있다.
- Python 리스트는 이름 있는 메서드만으로 다루지 않는다. `+`, 슬라이스, `del`, 슬라이스 대입처럼 시퀀스 문법으로 붙이고 자르고 바꾸는 경우가 많다.
- `join()`은 리스트가 아니라 문자열(str)의 메서드이며, 문자열 목록을 하나의 문자열로 만들 때 사용한다.
- Python 리스트에는 `splice()`라는 메서드가 없으므로, 구간 삭제와 구간 교체는 `del`, `insert()`, 슬라이스 대입으로 설명한다.
