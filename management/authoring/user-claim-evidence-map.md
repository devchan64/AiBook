# 사용자 관점과 유사 근거 매핑

## 목적

사용자가 제시한 AI 이해 관점을 외부 근거와 비교한다. 목적은 사용자의 표현을 그대로 증명하는 것이 아니라, 표준 자료에서 유사한 주장과 안전한 일반화 문구를 찾는 것이다.

## 사용자 관점

현재까지 정리된 핵심 관점은 다음과 같다.

- AI는 기존 소프트웨어의 함수 또는 요청 처리 방식을 다른 방식으로 구성한 것처럼 볼 수 있다.
- 사람이 직접 규칙을 작성하던 방식에서 데이터와 모델이 판단 기준을 맡는 방향으로 이동했다.
- AI 시스템은 사람이 수행하는 워크플로우와 유사하게, 요청을 받고 여러 단계를 거쳐 응답이나 행동을 만든다.
- 딥러닝은 직관적 추론의 방식을 컴퓨팅으로 해석한 것일 수 있다.

## 근거 후보 요약

근거 후보는 단순히 사용자의 주장과 비슷한지로만 채택하지 않는다. 본문 근거로 활용할 때는 다음 두 기준을 함께 본다.

- 학술적 일반화 가능성: 교과서, 학술 논문, 공식 문서, 반복 인용되는 연구처럼 넓은 설명으로 일반화할 수 있는가
- 중요도: 현재 Section의 중심 질문을 설명하는 핵심 근거인가, 아니면 보조적 비유나 주변 사례인가

## 핵심 근거 분석

### 1. Software 2.0

Karpathy의 `Software 2.0`은 사용자의 “AI는 SW의 함수를 무언가로 바꾼 것 같다”는 관점과 매우 가깝다.

이 글의 핵심은 다음처럼 요약할 수 있다.

- 전통적 소프트웨어는 사람이 명시적 명령을 코드로 작성한다.
- 신경망 기반 소프트웨어는 사람이 직접 가중치를 쓰는 대신, 데이터와 목표를 주고 학습으로 가중치를 찾는다.
- 훈련 데이터와 신경망 구조가 일종의 소스 역할을 하고, 학습 과정이 이를 실행 가능한 모델로 바꾼다.
- 복잡한 문제에서는 명시적 프로그램을 쓰는 것보다 원하는 행동을 데이터로 정의하는 편이 쉬울 수 있다.

본문에 반영 가능한 문장:

> 딥러닝 기반 모델은 사람이 모든 분기와 규칙을 코드로 쓰는 대신, 데이터와 목표를 통해 학습된 가중치가 입력을 처리하도록 만든다는 점에서 소프트웨어 작성 방식의 변화로 볼 수 있다.

주의:

- `Software 2.0`은 영향력 있는 비유이지만 표준 학술 정의는 아니다.
- 모든 AI를 `Software 2.0`으로 부르면 규칙 기반 AI, 탐색, 지식 표현, 확률 추론이 가려진다.

### 2. 연결주의와 인지 처리

Stanford Encyclopedia of Philosophy의 `Connectionism` 항목은 연결주의를 인공신경망으로 지적 능력을 설명하려는 인지과학 운동으로 설명한다. 또한 신경망이 뇌의 단순화된 모델이며, 단위와 가중치로 구성된다고 설명한다.

이 근거는 사용자의 “직관적 추론을 컴퓨팅으로 해석한 것”이라는 관점과 연결된다. 다만 `추론`이라는 단어는 조심해야 한다.

권장 보정:

> 딥러닝은 인간의 직관적 사고 전체를 구현한 것이 아니라, 명시 규칙으로 쓰기 어려운 인식, 분류, 유사성 판단, 패턴 처리를 다층 신경망의 표현 학습과 가중치 조정으로 계산 가능하게 만든 접근으로 볼 수 있다.

주의:

- 연결주의는 인간 인지를 설명하려는 계보와 관련이 있지만, 현대 딥러닝 모델이 인간의 직관 그 자체라는 뜻은 아니다.
- `reasoning`은 논리적 추론을 뜻할 수 있으므로, 이 문맥에서는 `직관적 인식`, `패턴 판단`, `인지 처리`가 더 안전하다.

### 3. 표현 학습과 다층 추상화

LeCun, Bengio, Hinton의 Nature review는 딥러닝이 여러 처리 층을 가진 계산 모델로 데이터의 여러 추상 수준 표현을 학습한다고 설명한다. 이 근거는 “직관적으로 보이는 판단을 계산 모델로 만든다”는 관점을 뒷받침한다.

본문에 반영 가능한 문장:

> 딥러닝의 핵심은 사람이 특징을 모두 설계하는 대신, 여러 층의 계산을 통해 데이터에서 더 추상적인 표현을 학습하게 만드는 것이다.

추가 보정 문장:

> 이 때문에 딥러닝은 사람이 직관적으로 처리하는 것처럼 보이는 이미지, 음성, 언어의 패턴 판단을 계산 가능한 표현 학습 문제로 옮긴 접근이라고 설명할 수 있다.

주의:

- `직관`을 직접 모델링했다기보다, 직관적으로 보이는 결과를 내는 패턴 판단을 데이터와 계산으로 학습한 것이다.
- 인간의 의식, 의도, 이해까지 설명한다고 확장하면 근거가 약해진다.

### 4. The Bitter Lesson

Sutton의 `The Bitter Lesson`은 AI 연구의 장기적 성과가 사람이 넣은 도메인 지식보다 계산을 활용하는 일반 방법에서 나왔다는 역사 해석을 제시한다. 특히 search와 learning을 반복해서 강조한다.

이 근거는 사용자의 “규칙에서 데이터 기반 판단으로 이동했다”는 관점과 연결된다.

본문에 반영 가능한 문장:

> AI의 발전은 사람이 지식을 직접 넣는 방식만으로는 한계가 있었고, 대규모 계산을 활용하는 탐색과 학습 방법이 여러 분야에서 더 강한 성과를 내는 방향으로 이동했다.

주의:

- Sutton의 글은 강한 역사 해석이며 모든 연구자가 같은 정도로 동의하는 표준 정의는 아니다.
- 기호 기반 AI나 명시적 지식의 역할이 사라졌다고 쓰면 안 된다.

### 5. MLOps와 워크플로우

Kreuzberger, Kühl, Hirschl의 MLOps 논문은 ML 제품을 생산 환경으로 가져가기 어렵고, 이를 위해 원칙, 구성요소, 역할, 아키텍처, workflows가 필요하다고 설명한다.

이 근거는 “AI 서비스는 워크플로우로 구성된다”는 관점과 연결되지만, 직접적으로 사용자 요청 처리 흐름을 설명하는 자료는 아니다. 운영 관점의 근거로 쓰는 것이 안전하다.

본문에 반영 가능한 문장:

> 실제 AI 서비스는 모델 하나로 끝나지 않고, 데이터 준비, 학습, 평가, 배포, 모니터링, 피드백을 포함하는 운영 워크플로우 안에서 유지된다.

### 6. Perception과 cognition의 분리

Sheth, Roy, Gaur의 `Neurosymbolic AI -- Why, What, and How`는 인간이 환경과 상호작용할 때 감각 입력을 상징으로 바꾸는 perception과, 그 상징을 지식에 연결해 추상화, 유추, 장기 계획을 지원하는 cognition을 함께 사용한다고 설명한다. 또한 AI에서 인간 perception에서 영감을 받은 machine perception을 신경망이 원시 데이터에서 수행하는 대규모 패턴 인식으로 설명한다.

이 근거는 사용자의 “딥러닝은 직관적 추론의 방식을 컴퓨팅으로 해석한 것 아닐까”라는 표현을 더 안전하게 나누는 데 유용하다.

본문에 반영 가능한 문장:

> 딥러닝은 인간의 사고 전체를 구현했다기보다, 사람이 직관적으로 처리하는 것처럼 보이는 감각적 인식, 분류, 유사성 판단, 언어 패턴 처리를 대규모 데이터 기반 패턴 인식과 표현 학습으로 계산 가능하게 만든 접근에 가깝다.

추론까지 포함해 쓰고 싶다면 다음처럼 제한한다.

> 딥러닝 모델의 inference는 학습된 표현과 가중치를 사용해 입력에 대한 출력을 계산하는 과정이며, 이 결과가 사람의 직관적 판단처럼 보일 수 있다. 그러나 명시적 근거를 따라가는 reasoning, 장기 계획, 규칙 검증은 별도의 상징적 표현, 검색, 도구, 검증 절차와 결합해 설명하는 편이 안전하다.

주의:

- `perception`과 `cognition`을 인간에게서 빌려오더라도, AI 모델이 인간과 같은 의식이나 이해를 가진다고 쓰면 안 된다.
- 딥러닝을 `reasoning` 자체로 정의하지 않는다. 패턴 인식과 표현 학습을 중심으로 쓰고, reasoning은 필요한 경우 symbolic 또는 neuro-symbolic 결합으로 분리한다.

## 현재 가장 안전한 일반화 문구

사용자의 주장을 본문용 문장으로 다듬으면 다음이 가장 안전하다.

> 딥러닝은 인간의 직관적 사고를 그대로 복제한 기술이라기보다, 사람이 명시적 규칙으로 쓰기 어려운 인식·분류·유사성 판단·언어 패턴 처리를 다층 신경망의 표현 학습과 가중치 최적화로 계산 가능하게 만든 접근으로 볼 수 있다.

조금 더 소프트웨어 관점으로 쓰면 다음 문장이 가능하다.

> 전통적인 소프트웨어가 사람이 작성한 규칙과 함수로 입력을 처리했다면, 딥러닝 기반 AI는 데이터와 목표를 통해 학습된 가중치 함수가 입력을 처리하도록 만든다. 이 점에서 딥러닝은 소프트웨어 작성 방식의 일부를 데이터 기반 최적화 문제로 바꾼 접근이라고 볼 수 있다.

워크플로우 관점은 다음처럼 분리한다.

> 현대 AI 서비스는 학습된 모델만이 아니라 입력 정리, 검색, 도구 호출, 규칙 검증, 모델 inference, 사람의 검토가 결합된 워크플로우로 구성된다.

다음 문장들은 본문에 직접 옮겨 쓸 수 있는 수준으로 정리한 일반화 문구다.

### 짧은 버전

> 딥러닝은 인간의 직관을 그대로 복제한 기술이라기보다, 명시적 규칙으로 쓰기 어려운 패턴 판단을 데이터, 표현 학습, 가중치 최적화로 계산 가능하게 만든 접근이다.

### 2.3 또는 역사 개론용

> 기호 기반 AI가 사람이 정리한 규칙과 지식 표현으로 문제를 풀려 했다면, 딥러닝은 사람이 규칙으로 다 쓰기 어려운 인식·분류·언어 패턴을 데이터에서 학습한 표현과 가중치로 처리하려는 흐름을 강하게 보여준다. 이 변화는 AI가 명시적 규칙만으로 닫히지 않는 문제를 다루기 위해 학습과 계산을 더 크게 활용하게 된 역사로 읽을 수 있다.

### 딥러닝 장용

> 딥러닝의 핵심은 여러 층의 계산을 통해 데이터의 표현을 학습하는 것이다. 낮은 층은 입력의 단순한 패턴을, 높은 층은 더 추상적인 패턴을 다룰 수 있다. 이 때문에 딥러닝은 사람이 직관적으로 알아보는 것처럼 보이는 이미지, 음성, 언어의 패턴 판단을 계산 가능한 학습 문제로 옮긴 접근이라고 설명할 수 있다.

### inference와 reasoning 구분용

> 딥러닝 모델의 inference는 학습된 파라미터를 사용해 입력에서 출력을 계산하는 과정이다. 이 출력이 사람의 직관적 판단처럼 보일 수는 있지만, 그것만으로 명시적 근거를 따라가는 reasoning이나 장기 계획을 보장하지는 않는다. 따라서 이 책에서는 `inference`, `직관적 패턴 판단`, `reasoning`을 구분해서 사용한다.

### AI 서비스 워크플로우용

> 현대 AI 서비스는 모델 하나가 단독으로 판단하는 구조라기보다, 입력 정리, 검색, 도구 호출, 규칙 검증, 모델 inference, 사람의 검토가 연결된 워크플로우로 구성된다. 모델은 이 워크플로우 안에서 사람이 규칙으로 쓰기 어려운 패턴 판단이나 생성 작업을 맡는 구성요소로 볼 수 있다.

### 사용자 관점 반영 문구

> 전통적 소프트웨어가 사람이 작성한 함수와 규칙으로 요청을 처리했다면, 딥러닝 기반 AI는 데이터와 목표를 통해 학습된 가중치 함수가 입력을 처리하도록 만든다. 이 점에서 딥러닝은 소프트웨어 작성 방식의 일부를 데이터 기반 최적화와 평가 문제로 바꾼 접근이라고 볼 수 있다.
