# Part 5. LLM과 생성형 AI

LLM과 생성형 AI 파트에서는 Transformer 이후의 흐름을 정리합니다. 모델 구조 자체뿐 아니라 실제 서비스를 만들 때 필요한 RAG, 도구 사용(tool use), 에이전트(agent), 평가, 운영 관점까지 함께 다룹니다.

## 다룰 내용

- 토큰화와 임베딩
- Attention과 Transformer
- BERT와 GPT 계열 모델의 차이
- 사전학습, 파인튜닝, 지시 튜닝
- 프롬프트 엔지니어링
- RAG
- 벡터 데이터베이스
- 도구 사용(tool use)과 에이전트(agent)
- MCP(Model Context Protocol)와 하네스(harness)
- LLM 애플리케이션 평가

## 완료 기준

- Transformer가 왜 LLM의 핵심 구조인지 설명할 수 있다.
- RAG가 필요한 상황과 한계를 설명할 수 있다.
- 에이전트(agent), MCP(Model Context Protocol), 하네스(harness)가 프롬프트와 다른 실행 환경의 주제임을 설명할 수 있다.
- 모델 API를 사용해 작은 생성형 AI 기능을 구현할 수 있다.
