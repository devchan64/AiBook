# Part 2. 기초 복구

Part 2는 머신러닝과 딥러닝을 다시 공부하기 전에 수학, Python, 데이터 도구, 문서 관리 감각을 회복하는 구간입니다. 여기서는 수학을 깊게 증명하거나 Python 문법을 모두 외우지 않습니다. 대신 이후 Part 3에서 모델 학습, 데이터 분리, 평가, 과적합, 일반화를 읽고 작은 코드로 확인할 수 있는 기반을 만듭니다.

이 파트의 핵심 목적은 “AI 문서와 예제 코드에서 반복해서 등장하는 계산 언어를 다시 읽을 수 있게 되는 것”입니다. 수식, 배열, 표, 그래프, 실행 환경, Git 이력은 서로 따로 떨어진 주제가 아닙니다. 모델 계산을 읽고, 데이터셋을 확인하고, 결과를 시각화하고, 학습 기록을 남기기 위해 함께 쓰이는 도구들입니다.

그래서 Part 2는 기초를 다시 연결합니다.

> 수식 표기 다시 읽기
> -> 선형대수, 미분, 확률, 통계, 최적화의 직관 복구
> -> Python 실행 환경과 문법 복습
> -> NumPy, Pandas, Matplotlib으로 작은 계산 확인
> -> Git으로 문서와 코드 변경 이력 관리
> -> Part 3 머신러닝으로 넘어가기 전 점검

## 이 파트의 목적

이 파트는 오래전에 배웠거나 부분적으로만 경험했던 수학과 소프트웨어 도구를 AI 재학습용 언어로 다시 정리하기 위한 구간입니다.

AI를 다시 공부할 때 수학이 막히는 이유는 반드시 수학 자체가 어려워서만은 아닙니다. \(x\), \(\sum\), \(\frac{1}{n}\), 벡터(vector), 행렬(matrix), 미분(derivative), 그래디언트(gradient), 확률(probability), 평균(mean), 손실(loss) 같은 표현이 코드, 데이터, 모델 설명 안에서 한꺼번에 등장하기 때문입니다.

또 Python, NumPy, Pandas, Matplotlib, Git도 각각 따로 배우면 흩어집니다. 하지만 머신러닝 문맥에서는 다음처럼 연결됩니다.

> Python으로 작은 계산을 실행한다.
> NumPy로 벡터와 행렬을 다룬다.
> Pandas로 표 형식 데이터를 읽는다.
> Matplotlib으로 데이터와 학습 흐름을 그린다.
> Git으로 원고, 코드, 이미지, 근거 메모의 변경 이력을 남긴다.

Part 2는 이 연결을 복구합니다. 목적은 전문가 수준의 수학 증명이나 소프트웨어 숙련이 아니라, Part 3의 머신러닝 설명을 읽으며 “이 계산이 무엇을 하고 있는지” 따라갈 수 있게 만드는 것입니다.

## 이 파트의 목표

Part 2를 읽고 나면 다음 정도의 이해를 갖는 것이 목표입니다.

- 변수(variable), 함수(function), 식(expression), 시그마(sigma), 극한(limit)을 AI 문서 독해용으로 다시 읽을 수 있습니다.
- 스칼라(scalar), 벡터(vector), 행렬(matrix), 벡터 공간(vector space), 행렬 곱(matrix multiplication)을 데이터와 모델 계산 관점으로 설명할 수 있습니다.
- 미분(derivative), 그래디언트(gradient), 손실 함수(loss function), 경사하강법(gradient descent)을 “값을 더 좋게 조정하기 위한 방향과 기준”으로 이해할 수 있습니다.
- 확률(probability), 분포(distribution), 평균(mean), 분산(variance), 표본(sample), 추정(estimation), 오차(error)를 데이터 판단의 기본 언어로 사용할 수 있습니다.
- Python 실행 환경, 터미널, 가상환경, 의존성, 노트북 실행 흐름을 구분할 수 있습니다.
- Python의 값, 변수, 리스트, 딕셔너리, 반복, 함수, 클래스의 입문적 역할을 설명할 수 있습니다.
- NumPy 배열(array), shape, axis, indexing, slicing, broadcasting, vectorization을 모델 계산과 연결할 수 있습니다.
- Pandas DataFrame을 행(row), 열(column), 인덱스(index)로 읽고, 데이터셋 준비의 직관을 가질 수 있습니다.
- Matplotlib으로 수식, 분포, 관계, 손실 곡선을 시각적으로 확인할 수 있습니다.
- Git 커밋과 브랜치를 학습 문서의 재현성 관리 도구로 이해할 수 있습니다.

## 무엇을 설명하는가

Part 2는 15개 Chapter로 구성됩니다.

먼저 수학을 계산 언어로 다시 읽습니다. 변수, 함수, 식, 시그마, 극한을 복구하고, 수식을 코드와 데이터에서 다시 확인하는 관점을 잡습니다.

그다음 모델 계산을 읽기 위한 핵심 수학을 다룹니다. 선형대수에서는 스칼라, 벡터, 행렬, 벡터 공간, 행렬 곱을 봅니다. 미분에서는 변화율, 기울기, 그래디언트, 학습에서 미분이 필요한 이유를 봅니다. 확률과 통계에서는 불확실성을 숫자로 표현하고, 데이터 묶음의 모양을 평균, 분산, 분포, 표본, 오차로 읽는 법을 정리합니다. 최적화에서는 좋은 답을 직접 쓰는 것이 아니라 더 나은 값을 찾는 문제로 접근합니다.

중간에서는 실행 환경과 Python 기초를 다룹니다. Colab과 로컬 PC, 터미널과 셸, Python 인터프리터와 스크립트, 가상환경과 패키지, 의존성과 재현성을 구분합니다. 이어서 Python의 값, 변수, 타입, 리스트, 딕셔너리, 반복, 함수, 클래스의 기본 감각을 복구합니다.

후반부에서는 자료구조와 데이터 도구를 다룹니다. 배열, 표, 트리, 그래프의 직관을 잡고, Jupyter와 Colab 노트북으로 코드와 설명을 함께 남기는 방법을 봅니다. NumPy로 벡터와 행렬을 계산하고, Pandas로 표 형식 데이터를 다루며, Matplotlib으로 숫자의 모양을 확인합니다.

마지막으로 Git과 문서 관리를 다룹니다. 원고, 예제 코드, 이미지, 근거 메모, 배포 목차가 함께 바뀌는 프로젝트에서는 변경 이력과 재현성이 중요합니다. Part 2의 끝에서는 수식을 코드로 옮기는 작은 절차를 정리하고, Part 3로 넘어가기 전에 필요한 감각을 점검합니다.

## 왜 필요한가

머신러닝을 공부할 때 많은 설명은 다음 형태로 나타납니다.

> 입력 데이터 \(X\)가 있고, 라벨 \(y\)가 있다.
> 모델은 예측값 \(\hat{y}\)를 만든다.
> 손실 함수는 예측과 실제값의 차이를 숫자로 만든다.
> 학습은 그 손실을 줄이는 방향으로 파라미터를 조정한다.
> 평가는 보지 못한 데이터에서 결과를 확인한다.

이 설명을 읽으려면 여러 기반이 동시에 필요합니다. \(X\)와 \(y\)를 배열과 표로 볼 수 있어야 하고, 손실을 평균과 합산으로 읽을 수 있어야 하며, 그래프를 보고 학습 흐름을 점검할 수 있어야 합니다. 또한 예제 코드가 어느 환경에서 실행되는지, 결과 이미지와 원고가 어떤 커밋에서 함께 바뀌었는지도 추적할 수 있어야 합니다.

Part 2는 이 기반을 과도하게 깊게 파지 않습니다. 대신 Part 3에서 모델과 데이터 흐름을 읽다가 막히지 않도록 최소한의 언어와 실습 감각을 마련합니다.

## 이 파트를 마치면 생기는 이해

이 파트를 마치면 수학과 소프트웨어 도구를 따로따로 외우는 대신, 하나의 학습 흐름으로 볼 수 있습니다.

> 수식은 계산의 의도를 적는다.
> Python은 작은 계산을 실행한다.
> NumPy는 벡터와 행렬 계산을 재사용한다.
> Pandas는 데이터셋을 표로 읽게 한다.
> Matplotlib은 숫자의 모양을 확인하게 한다.
> Git은 원고와 코드와 결과의 변경 이력을 남긴다.

이 이해가 생기면 Part 3의 머신러닝에서 `X`, `y`, `fit`, `predict`, loss, metric, train, validation, test 같은 표현을 완전히 낯선 말로 보지 않게 됩니다. Part 2는 모든 기초를 끝내는 구간이 아니라, 머신러닝 학습으로 넘어갈 수 있는 최소한의 공통 기반을 만드는 구간입니다.

## 완료 기준

- 수식에서 변수, 함수, 시그마, 평균, 오차가 어떤 계산 절차로 이어지는지 설명할 수 있다.
- 벡터와 행렬이 데이터와 모델 계산에서 어디에 쓰이는지 말할 수 있다.
- 미분, 그래디언트, 손실 함수, 경사하강법의 역할을 입문 수준으로 설명할 수 있다.
- 확률, 분포, 평균, 분산, 표본, 추정을 데이터 판단의 언어로 구분할 수 있다.
- Python 코드가 어디에서 실행되는지, Colab과 로컬 PC의 차이가 무엇인지 설명할 수 있다.
- Python 리스트, 딕셔너리, 반복, 함수, 클래스의 기본 역할을 말할 수 있다.
- NumPy 배열의 shape, axis, indexing, slicing, broadcasting을 작은 예제로 확인할 수 있다.
- Pandas DataFrame을 행과 열 중심으로 읽고, 학습용 데이터셋 준비의 필요성을 설명할 수 있다.
- Matplotlib으로 함수 모양, 산점도, 히스토그램, 손실 곡선을 확인할 수 있다.
- Git 커밋과 브랜치를 원고, 코드, 이미지, 근거 메모의 변경 이력 관리 관점으로 설명할 수 있다.
- Part 3로 넘어가기 전에 `X`, `y`, sample, feature, fit, predict의 기본 의미를 말할 수 있다.
