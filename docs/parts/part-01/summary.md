# Part 1 마무리. AI 개론과 지형도 정리

Part 1은 AI를 다시 공부하기 전에 전체 지형을 잡기 위한 구간이었습니다. 세부 알고리즘을 구현하거나 수식을 깊게 증명하기보다, 이후 학습에서 반복해서 만나게 될 용어와 관점을 먼저 정리했습니다.

이 Part에서 가장 중요한 목표는 AI를 하나의 유행어나 하나의 제품 이름으로 보지 않는 것입니다. AI는 규칙 기반 접근, 탐색, 휴리스틱, 확률적 판단, 데이터 기반 학습, 딥러닝, 생성형 AI, LLM, 에이전트, 서비스 운영, 사회적 책임이 겹쳐 있는 넓은 분야입니다.

## 이 Part에서 다룬 핵심 흐름

Part 1의 흐름은 다음처럼 정리할 수 있습니다.

> AI라는 말의 범위
> -> 규칙 기반 접근과 기호 기반 AI
> -> 탐색, 지식 표현, 확률 추론
> -> 데이터에서 패턴을 배우는 머신러닝
> -> 표현을 학습하는 딥러닝
> -> 생성형 AI와 LLM
> -> 프롬프트, 임베딩, RAG, 에이전트
> -> 서비스 아키텍처와 운영 제약
> -> 윤리, 저작권, 보안, 실무 적용, 전망 검토

이 흐름은 “옛 기술이 사라지고 새 기술이 완전히 대체했다”는 뜻이 아닙니다. 시대에 따라 중심이 되는 접근이 달라졌고, 어떤 개념은 이름과 위치를 바꾸어 지금도 계속 사용됩니다.

예를 들어 탐색(search)과 휴리스틱(heuristic)은 초기 AI의 문제 해결 방식에서 중요했지만, 현대 AI에서도 후보를 줄이고 계산 비용을 낮추는 사고방식으로 다시 나타납니다. 확률(probability)과 불확실성(uncertainty)은 예측, 분류, 생성 결과를 이해할 때 계속 등장합니다.

## 반드시 기억할 개념

Part 1에서 가장 오래 가져가야 할 개념은 구분입니다.

| 구분 | 기억할 관점 |
| --- | --- |
| AI와 머신러닝 | AI는 더 넓은 분야이고, 머신러닝은 데이터에서 패턴을 배우는 접근입니다. |
| 머신러닝과 딥러닝 | 딥러닝은 신경망을 이용해 표현을 학습하는 머신러닝의 중요한 흐름입니다. |
| 딥러닝과 LLM | LLM은 딥러닝의 여러 발전 중 언어 모델링과 Transformer 계열에서 강하게 성장한 흐름입니다. |
| 학습과 모델 실행 | 학습(learning)은 모델 내부 값을 바꾸는 과정이고, 모델 실행(inference)은 학습된 모델을 사용하는 과정입니다. |
| 프롬프트와 모델 | 프롬프트(prompt)는 모델의 동작을 지정하는 입력이지, 모델 내부 지식이나 동작을 완전히 통제하는 장치가 아닙니다. |
| RAG와 에이전트 | RAG는 외부 정보를 검색해 입력에 보태는 구조이고, 에이전트(agent)는 목표를 여러 작업 흐름으로 이어 가는 구조입니다. |
| 모델과 서비스 | 실제 AI 서비스는 모델 하나가 아니라 앱, 데이터, 도구, 권한, 로그, 평가, 운영 제약이 함께 작동하는 구조입니다. |

이 구분이 잡히면 이후 Part에서 수학, 코드, 모델, 데이터, 서비스 구조를 배울 때 현재 개념이 전체 지도에서 어디에 놓이는지 확인할 수 있습니다.

## 오해하기 쉬운 지점

Part 1에서 특히 조심해야 할 오해는 다음입니다.

- AI를 LLM과 같은 말로 보지 않습니다.
- 생성형 AI를 모든 AI의 최종 형태처럼 설명하지 않습니다.
- 규칙 기반 접근을 낡아서 사라진 방식으로만 보지 않습니다.
- 휴리스틱을 확률 모델과 같은 것으로 섞지 않습니다.
- 라벨(label)을 정답이라는 말로만 이해하지 않습니다.
- 임베딩(embedding)을 사람이 직접 읽을 수 있는 의미 정의로 오해하지 않습니다.
- 프롬프트(prompt)를 모델을 완전히 제어하는 명령문으로 오해하지 않습니다.
- 에이전트(agent)를 무제한 자율 실행 시스템으로 단정하지 않습니다.
- AI 전망을 개인적 추측이나 AI가 만든 문장만으로 작성하지 않습니다.

이 책에서는 이런 오해를 줄이기 위해 한국어 표현과 영어 원어를 함께 확인합니다. 특히 `추론`, `모델`, `파라미터`, `생성`, `층위`처럼 문맥에 따라 의미가 달라지는 말은 계속 구분해야 합니다.

## 다음 Part로 넘어가기 전에 확인할 질문

Part 2로 넘어가기 전에 다음 질문에 답할 수 있으면 충분합니다.

- AI, 머신러닝(machine learning), 딥러닝(deep learning), 생성형 AI(generative AI), LLM(large language model)의 관계를 큰 그림으로 설명할 수 있는가?
- 규칙 기반 접근(rule-based approach)과 학습 기반 접근(learning-based approach)의 차이를 말할 수 있는가?
- 데이터(data), 특징(feature), 표현(representation), 모델(model), 파라미터(parameter), 학습(learning), 모델 실행(inference)의 관계를 대략 설명할 수 있는가?
- 불확실성(uncertainty), 확률(probability), 확률적 과정(stochastic)이 같은 말이 아님을 설명할 수 있는가?
- 프롬프트(prompt), 임베딩(embedding), 벡터 검색(vector search), RAG, 에이전트(agent)가 서로 다른 역할을 가진다는 점을 설명할 수 있는가?
- AI 서비스가 모델 하나만으로 만들어지지 않는다는 점을 설명할 수 있는가?
- 개인적 작업 가설과 외부 근거가 있는 사실 주장을 구분할 수 있는가?

이 질문에 모두 완벽하게 답할 필요는 없습니다. Part 1의 목적은 정답을 암기하는 것이 아니라, 이후 학습에서 다시 돌아올 지도를 만드는 것입니다.

## 다음 Part와 연결되는 관점

Part 2에서는 기초 복구로 넘어갑니다. 여기서 말하는 기초는 수학을 깊게 증명하는 능력이 아니라, AI 문서와 코드를 읽기 위한 최소 언어입니다.

Part 1에서 다룬 모델, 데이터, 표현, 파라미터, 학습, 모델 실행은 Part 2에서 수식과 코드로 다시 나타납니다.

> 모델이 입력을 받는다.
> 입력은 숫자, 벡터, 행렬로 표현된다.
> 학습은 값을 조정한다.
> 조정은 손실과 최적화의 언어로 설명된다.
> 확률과 통계는 불확실한 데이터를 다루는 언어가 된다.
> 코드는 이 계산을 재현하는 도구가 된다.

따라서 Part 2는 AI를 수학으로 어렵게 만들기 위한 구간이 아닙니다. Part 1에서 만든 지도를 실제 계산 언어로 조금씩 확인하기 위한 구간입니다.

## Part 1을 마치며

Part 1을 마쳤다면 AI를 하나의 단어가 아니라 여러 층위(level)의 개념으로 볼 준비가 된 것입니다.

이제 중요한 태도는 두 가지입니다.

첫째, 개인적 직관을 버리지 않습니다. 오래전에 배운 기억, 실무에서 느낀 의문, 최근 도구를 쓰며 생긴 감각은 학습의 좋은 출발점입니다.

둘째, 그 직관을 사실처럼 단정하지 않습니다. 표준 용어, 외부 근거, 실제 코드, 작은 실습을 통해 검토하면서 이해를 조정합니다.

Part 2부터는 이 태도를 유지한 채 수학, 코드, 데이터 계산을 다시 회복합니다.
