# 기준 목차

이 문서는 책의 기준 목차입니다. 실제 장 제목과 순서는 학습을 진행하면서 바뀔 수 있지만, 큰 흐름은 유지합니다.

현재 기준 흐름은 다음과 같습니다.

```mermaid
flowchart LR
  A[AI 개론과 지형도]
  B[기초 복구]
  C[머신러닝]
  D[딥러닝]
  E[LLM과 생성형 AI]
  F[프로젝트]

  A --> B --> C --> D --> E --> F
```

## 문서 상태

- 상태: 초안
- 목적: 책의 큰 구조와 확장 기준 정의
- 적용 범위: `docs/` 원고와 `mkdocs.yml`의 `nav`
- 변경 원칙: 참고 목차를 그대로 복사하지 않고 책의 목적에 맞게 재배치

## 설계 원칙

- 먼저 커리큘럼의 기준을 잡고, 그 기준에 맞춰 자료를 수집합니다.
- AI는 자료 수집, 정리, 초안 작성, 비교, 다이어그램 작성을 돕습니다.
- 사람은 방향 설정, 오류 검토, 해석 보정, 최종 판단을 담당합니다.
- 개인적인 작업 가설과 표준적인 설명을 구분합니다.
- 틀린 내용이 발견되면 감추지 않고 정정합니다.
- 개론에서 모든 것을 깊게 설명하지 않습니다. 개론은 뒤에서 다시 배울 내용을 연결하는 지도 역할을 합니다.
- 각 장은 가능하면 `목표`, `핵심 개념`, `예시 또는 실습`, `체크리스트`, `출처와 참고 자료`를 포함합니다.

## 시작 문서

시작 문서는 책의 목적, 작성 방식, 운영 원칙을 설명합니다.

| 문서 | 역할 |
| --- | --- |
| 소개 | 책의 목적과 전체 방향 |
| 머리말 | 대상 독자와 읽는 방법 |
| 기준 목차 | 커리큘럼 기준과 확장 계획 |
| Codex 소개와 사용 원칙 | 이 책을 만드는 AI 에이전트 도구 설명 |
| 시각화 작성 원칙 | 차트와 다이어그램 작성 기준 |

## 로드맵

### 1장. AI 재학습 로드맵

역할: 책 전체의 방향을 잡는 장입니다.

다룰 질문:

- 왜 다시 배우는가
- AI, 머신러닝, 딥러닝, 생성형 AI는 어떻게 연결되는가
- 망각한 기초를 어떤 순서로 복구할 것인가
- 개인적인 작업 가설과 표준적인 설명을 어떻게 구분할 것인가
- AI 도구로 만든 책의 오류 가능성을 어떻게 다룰 것인가

## Part 1. AI 개론과 지형도

역할: 세부 기술을 깊게 들어가기 전에 전체 지형을 다시 잡습니다. "AI란 무엇인가"에서 시작하되, 단순 입문서식 나열이 아니라 이후 파트로 이어지는 연결 지도를 만듭니다.

장 후보:

- AI란 무엇인가
- 규칙 기반 AI와 학습 기반 AI
- 데이터, 특징, 모델
- 지도학습, 비지도학습, 강화학습
- 딥러닝 개론
- 생성형 AI 개론
- LLM과 프롬프트
- 임베딩과 벡터 검색
- AI 서비스 아키텍처
- AI 윤리, 저작권, 보안
- 실무 적용 사례
- 앞으로의 AI

완료 기준:

- AI라는 말이 너무 넓게 쓰일 때 생기는 혼동을 설명할 수 있다.
- 규칙 기반 접근과 학습 기반 접근의 차이를 설명할 수 있다.
- 데이터, 특징, 모델의 관계를 설명할 수 있다.
- LLM, 임베딩, 벡터 검색, RAG가 어떻게 연결되는지 큰 흐름을 말할 수 있다.

## Part 2. 기초 복구

역할: 머신러닝과 딥러닝을 다시 읽고 구현할 수 있는 최소 기반을 회복합니다.

장 후보:

- 학습 환경 세팅
- Python 핵심 문법 복습
- Jupyter, Colab, 가상환경
- NumPy
- Pandas
- Matplotlib과 기본 시각화
- Git과 문서 관리
- 선형대수 핵심
- 미분과 편미분
- 확률과 통계
- 최적화와 경사하강법

완료 기준:

- 간단한 데이터셋을 불러와 탐색할 수 있다.
- 벡터와 행렬 연산이 모델 계산에서 어디에 쓰이는지 설명할 수 있다.
- 손실 함수와 경사하강법의 역할을 설명할 수 있다.

## Part 3. 머신러닝

역할: 데이터로부터 규칙을 학습한다는 말의 의미를 복구합니다. 딥러닝으로 넘어가기 전에 학습, 일반화, 평가, 과적합을 분명히 잡습니다.

장 후보:

- AI, 머신러닝, 딥러닝의 구분
- 지도학습, 비지도학습, 강화학습
- 데이터 분리와 검증
- 과적합과 일반화
- 평가 지표
- 선형회귀
- 로지스틱 회귀
- k-NN
- SVM
- 결정트리
- 랜덤포레스트
- 그래디언트 부스팅
- 클러스터링
- 차원 축소

완료 기준:

- 문제 유형에 맞는 모델과 평가 지표를 고를 수 있다.
- 과적합과 일반화를 설명할 수 있다.
- scikit-learn으로 기본 학습 파이프라인을 구성할 수 있다.

## Part 4. 딥러닝

역할: 신경망이 데이터를 어떻게 표현하고 학습하는지 이해합니다. 특히 가중치, 최적화, 병렬 처리, 표현 학습이 왜 중요한지 검토합니다.

장 후보:

- 퍼셉트론
- 다층 신경망
- 활성화 함수
- 손실 함수
- 역전파
- 옵티마이저
- 정규화와 드롭아웃
- GPU와 병렬 처리
- CNN
- RNN, LSTM, GRU
- Attention
- Transformer

완료 기준:

- 순전파와 역전파의 흐름을 설명할 수 있다.
- 가중치 기반 표현이 명시적 규칙과 어떻게 다른지 설명할 수 있다.
- GPU 병렬 처리가 딥러닝 발전에 어떤 역할을 했는지 설명할 수 있다.

## Part 5. LLM과 생성형 AI

역할: Transformer 이후의 흐름을 이해하고, LLM을 실제 서비스와 연결합니다. 프롬프트, 임베딩, RAG, Agent를 각각 따로 보지 않고 하나의 시스템 흐름으로 정리합니다.

장 후보:

- 토큰화
- 임베딩
- Transformer 구조 복습
- BERT 계열
- GPT 계열
- 사전학습
- 파인튜닝
- 지시 튜닝과 정렬
- 프롬프트 엔지니어링
- RAG
- 벡터 데이터베이스
- Tool use
- Agent
- LLM 평가
- 비용, 지연 시간, 운영 고려사항

완료 기준:

- LLM이 다음 토큰 예측 모델이라는 설명의 의미와 한계를 말할 수 있다.
- 프롬프트, 임베딩, 벡터 검색, RAG의 관계를 설명할 수 있다.
- Agent가 단순 챗봇과 어떻게 다른지 설명할 수 있다.

## Part 6. 프로젝트

역할: 배운 내용을 작은 산출물로 검증합니다. 프로젝트는 학습 내용을 실제 작동하는 형태로 연결하고, 한계와 개선점을 기록하는 방식으로 작성합니다.

프로젝트 후보:

- 데이터 분석 미니 프로젝트
- 전통 머신러닝 예측 모델
- 이미지 분류 모델
- 텍스트 분류 모델
- 문서 기반 RAG 챗봇
- 도구를 사용하는 Agent
- 배포와 모니터링

완료 기준:

- 문제 정의, 데이터, 접근 방법, 구현, 결과, 한계가 문서에 남아 있다.
- 프로젝트가 앞선 파트의 어떤 개념을 검증하는지 연결되어 있다.
- 실패하거나 부족한 부분도 기록되어 있다.

## 검증 필요 목록

다음 주제는 개인적인 이해와 표준적인 설명의 차이를 확인하면서 작성합니다.

| 주제 | 확인할 질문 |
| --- | --- |
| 불확정성과 확률적 사고 | 딥러닝 이후 AI 연구에서 불확정성은 어떤 의미로 쓰이는가 |
| 규칙 기반 AI와 학습 기반 AI | 명시적 규칙과 통계적 학습의 차이를 어떻게 설명할 것인가 |
| 가중치 기반 표현 | 가중치가 지식, 추론, 일반화와 어떤 관계를 갖는가 |
| GPU와 병렬 처리 | 병렬 처리가 딥러닝 발전에 어떤 역할을 했는가 |
| LLM의 불확실성 | LLM의 출력 불확실성을 어떻게 이해하고 평가할 것인가 |
| 윤리, 저작권, 보안 | 입문 수준에서 어디까지 다룰 것인가 |
| AI 서비스 아키텍처 | 개론에서 먼저 소개할지, 프로젝트 파트에서 자세히 다룰지 |
| CLI에서 LLM 에이전트로 | LLM 에이전트를 새로운 인터페이스 계층으로 볼 수 있는가 |
| Codex와 OS | Codex 같은 에이전트 도구는 OS를 대체하는가, OS 위의 작업 인터페이스를 확장하는가 |

## 확장 규칙

새 장을 추가할 때는 다음을 확인합니다.

- 어느 Part에 속하는가
- 앞 장의 어떤 개념을 이어받는가
- 뒤 장의 어떤 개념을 준비하는가
- 자체 초안인지, 외부 자료 기반인지
- `mkdocs.yml`의 `nav`에 반영해야 하는가
- 차트나 다이어그램이 필요한가
- 검증 필요 목록에 추가할 질문이 있는가

## 출처와 참고 자료

이 문서는 현재 프로젝트의 기준 목차와 집필 방향을 정리한 자체 초안이며, 외부 자료를 직접 인용하지 않았습니다.
