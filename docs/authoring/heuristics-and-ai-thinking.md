# 휴리스틱과 AI적 사고

이 문서는 휴리스틱을 AI 재학습 목차에서 어떻게 설명할지 정리합니다. 목적은 휴리스틱을 단순한 실무 팁으로 축소하지 않고, 초기 AI의 문제 해결 방식과 불확실한 상황에서의 판단 방식으로 일반화하는 것입니다.

## 사용자 직관

사용자의 작업 가설은 다음과 같습니다.

> 휴리스틱은 AI적 사고를 만들기 위한 인류의 첫 시도에 가깝지 않을까. 휴리스틱은 불확실성을 반영하고자 한 시도라고 볼 수 있지 않을까.

이 직관은 유용하지만, 그대로 단정하면 과장될 수 있습니다. 더 안전한 표현은 다음과 같습니다.

> 휴리스틱은 초기 AI가 인간의 문제 해결을 모방하려 할 때 사용한 핵심 방법 중 하나였다. 완전한 탐색이나 최적화가 어려운 상황에서, 제한된 정보와 계산 자원으로 충분히 좋은 방향을 선택하게 하는 규칙으로 이해할 수 있다.

또 다른 일반화는 다음과 같습니다.

> 휴리스틱은 불확실성과 계산 한계를 소프트웨어의 선택 규칙으로 반영하려는 시도였다.

확률과 현대 AI에 대한 작업 가설은 다음처럼 정리합니다.

> 인간은 알 수 없는 현상을 확률로 풀어보려 했고, 현대 AI는 확률적 예측과 선택이 대규모로 반복될 때 인간 사고와 비슷해 보이는 행동이 나타날 수 있음을 보여주는 사례일 수 있다.

이 표현은 반드시 `작업 가설`로 둡니다. 표준 설명으로 바꿀 때는 "인간의 사고와 같다"가 아니라 "인간의 언어 사용, 판단, 생성 행동과 비슷한 출력이 나타난다"로 제한합니다.

## 전달할 핵심 일반화

이 책에서 전달하려는 핵심 일반화는 다음입니다.

> 인간은 오래전부터 불확실한 현상을 이해하고 다루려 했다. 주사위와 도박은 우연을 경험하고 계산하려는 출발점이었고, 확률은 그 우연을 수학으로 표현하려는 시도였다. 휴리스틱은 모든 경우를 계산할 수 없을 때 충분히 좋은 선택을 하려는 사고 방식이며, 초기 AI는 이 방식을 소프트웨어의 탐색 규칙으로 구현하려 했다. 현대 AI는 대규모 데이터에서 확률적 패턴을 학습하고 반복적으로 예측함으로써, 인간의 사고와 비슷해 보이는 언어와 판단 행동을 만들어낸다.

더 짧게 말하면 다음과 같습니다.

> AI의 한 흐름은 불확실한 세계를 확률과 경험 규칙으로 다루려는 인간의 시도가 소프트웨어와 학습 모델로 옮겨온 역사다.

이 문장은 책의 도입부에서 사용할 수 있지만, 반드시 다음 제한을 함께 둡니다.

- 휴리스틱은 AI의 유일한 출발점이 아닙니다.
- 확률적 예측이 반복된다고 인간의 사고 자체가 생긴다고 단정하지 않습니다.
- 관찰할 수 있는 것은 인간과 비슷해 보이는 출력, 판단, 문제 해결 행동입니다.
- 그 유사성이 어디까지 표면적 행동이고 어디부터 추론 능력인지 별도로 검증해야 합니다.

## 표준적 설명

- 휴리스틱은 모든 경우에 최적해를 보장하는 공식이 아니라, 탐색이나 판단을 유망한 방향으로 이끄는 경험적 규칙입니다.
- 초기 AI에서는 문제 해결을 탐색(search)으로 보고, 탐색 공간을 줄이기 위해 휴리스틱을 사용했습니다.
- Newell과 Simon의 작업은 인간 문제 해결을 컴퓨터로 모사하려는 시도와 연결됩니다. ACM의 Newell 소개는 Logic Theorist와 General Problem Solver가 문제 해결에 휴리스틱을 사용했다고 설명합니다.
- AIMA 목차에서도 informed heuristic search, heuristic functions, learning heuristics from experience가 AI 문제 해결의 주요 항목으로 배치되어 있습니다.
- Herbert Simon의 bounded rationality와 satisficing은 완전한 최적화 대신 제한된 정보와 계산 능력 안에서 충분히 만족스러운 해를 찾는 관점입니다.
- Kahneman과 Tversky의 연구는 사람이 불확실한 조건에서 휴리스틱을 사용하지만, 그 결과가 체계적 편향으로 이어질 수 있음을 보여주는 흐름입니다.

## 일반화

이 책에서는 휴리스틱을 다음처럼 일반화합니다.

| 관점 | 설명 |
| --- | --- |
| 계산 관점 | 가능한 모든 경우를 다 살펴볼 수 없을 때 탐색을 줄이는 규칙 |
| 소프트웨어 관점 | 불확실성과 계산 한계를 프로그램 안의 선택 규칙, 평가 함수, 우선순위로 옮기는 방식 |
| 인지 관점 | 인간이 제한된 정보와 시간 안에서 판단하는 방식 |
| 불확실성 관점 | 정답, 확률, 결과를 완전히 알 수 없을 때 방향을 정하는 임시 기준 |
| 실무 관점 | 데이터 전처리, 모델 선택, 튜닝에서 검증 가능한 출발점 |
| 위험 관점 | 빠른 판단을 돕지만 편향이나 과신을 만들 수 있음 |

## 불확실성과의 관계

휴리스틱이 불확실성을 반영한다는 표현은 부분적으로 맞습니다. 다만 확률 모델처럼 불확실성을 명시적으로 계산하는 방식과는 구분해야 합니다.

| 구분 | 설명 |
| --- | --- |
| 휴리스틱 | 불확실하거나 너무 큰 문제에서 방향을 정하는 경험적 규칙 |
| 확률 모델 | 불확실성을 확률 분포나 조건부 확률로 명시하는 모델 |
| 최적화 | 명확한 목적 함수를 두고 더 좋은 해를 찾는 과정 |
| 학습 | 데이터에서 규칙이나 표현을 조정하는 과정 |

따라서 휴리스틱은 불확실성을 수학적으로 표현하는 방법이라기보다, 불확실성과 계산 한계가 있는 상황에서 판단을 가능하게 하는 실용적 장치로 설명합니다.

소프트웨어 관점에서는 휴리스틱을 `if-then` 규칙, 점수 함수, 우선순위 큐, 탐색 중단 조건, 모델 선택 기준처럼 구현할 수 있습니다. 이때 프로그램은 세계의 불확실성을 완전히 이해하는 것이 아니라, 불확실한 상황에서도 다음 행동을 고를 수 있도록 제한된 기준을 갖게 됩니다.

## 주사위 게임과 확률

주사위 게임이나 도박도 넓게 보면 불확실성을 다루려는 인류의 오래된 시도로 볼 수 있습니다. 다만 휴리스틱과 같은 층위는 아닙니다.

| 층위 | 질문 | 대표 형태 |
| --- | --- | --- |
| 놀이와 도박 | 결과를 알 수 없는 상황에서 어떻게 행동하는가 | 주사위, 카드, 내기 |
| 확률 | 가능한 결과가 얼마나 그럴듯한가 | 경우의 수, 기대값, 확률 분포 |
| 휴리스틱 | 모든 계산이 불가능할 때 어떤 선택을 먼저 할 것인가 | 경험 규칙, 평가 함수, 탐색 우선순위 |
| AI 시스템 | 불확실성과 계산 한계를 어떻게 프로그램으로 다룰 것인가 | 탐색 알고리즘, 확률 모델, 학습 모델 |

이 흐름으로 보면 주사위 게임은 "AI적 사고"의 직접 시작이라기보다, 불확실성을 관찰하고 수량화하려는 훨씬 오래된 인간의 시도입니다. 그 뒤 확률론은 불확실성을 수학으로 다루는 방법을 만들었고, 휴리스틱은 불확실성과 계산 한계를 소프트웨어의 선택 규칙으로 다루는 방법이 되었습니다.

사용자의 표현을 일반화하면 다음과 같습니다.

> 주사위 게임은 불확실한 현상을 확률로 풀어보려는 초기 사례이고, 현대 AI는 확률적 예측을 대규모로 반복해 인간과 비슷해 보이는 판단 또는 생성을 만들어내는 사례로 볼 수 있다.

이 설명에서 조심할 점은 두 가지입니다.

- 주사위의 확률은 가능한 결과가 비교적 명확한 불확실성입니다.
- 현대 AI의 확률적 예측은 데이터, 모델 구조, 손실 함수, 학습 과정이 결합된 결과이며 단순한 주사위 반복과 같지 않습니다.

따라서 책에서는 다음 흐름으로 설명합니다.

```mermaid
flowchart LR
  A[주사위와 우연의 경험]
  B[확률과 기대값]
  C[휴리스틱 탐색]
  D[확률적 AI와 머신러닝]
  E[불확실한 상황에서의 AI 시스템]
  F[인간과 비슷해 보이는 출력과 판단]

  A --> B --> C --> D --> E --> F
```

## 책에서의 배치

휴리스틱은 한 파트에만 가두지 않습니다.

1. Part 1에서는 AI 역사와 문제 해결의 관점에서 다룹니다.
2. Part 3에서는 머신러닝 실무의 경험 규칙으로 다룹니다.
3. 프로젝트 파트에서는 휴리스틱을 사용하되, 반드시 검증 데이터와 평가 지표로 확인합니다.

## 주의할 표현

| 피할 표현 | 이유 | 대체 표현 |
| --- | --- | --- |
| 휴리스틱은 AI의 첫 시도다 | 논리, 탐색, 규칙 기반 AI 등 여러 초기 흐름이 함께 존재함 | 휴리스틱은 초기 AI의 핵심 문제 해결 방식 중 하나다 |
| 휴리스틱은 불확실성을 계산한다 | 확률 모델과 혼동됨 | 휴리스틱은 불확실한 상황에서 판단 방향을 잡는 규칙이다 |
| 휴리스틱은 좋은 실무 팁이다 | 역사적, 인지적 의미가 사라짐 | 휴리스틱은 탐색, 판단, 모델링의 경험적 기준이다 |
| 휴리스틱은 정답이다 | 편향과 실패 가능성을 숨김 | 휴리스틱은 검증해야 할 출발점이다 |
| 확률이 반복되면 인간 사고가 된다 | 행동 유사성과 사고의 동일성을 혼동함 | 확률적 예측의 반복이 인간과 비슷해 보이는 출력을 만들 수 있다 |

## 목차 반영 원칙

- `휴리스틱이 필요한 이유` 장에서는 인간 문제 해결, 탐색 공간, 계산 한계, bounded rationality를 함께 소개합니다.
- 머신러닝 파트에서는 특징 선택, 전처리, 모델 선택, 하이퍼파라미터 튜닝의 경험 규칙을 다룹니다.
- 휴리스틱을 설명할 때는 반드시 실패 가능성도 함께 설명합니다.
- 휴리스틱을 사용한 판단은 검증 데이터, 평가 지표, 재현 가능한 실험으로 확인합니다.
